# 模块 07 — 内容审核

> 路径: `moderation/` | URL前缀: `/moderation/` | 独立应用模块（不在 `apps/` 下）

---

## 功能概览

全站内容审核与用户信誉管理中。支持敏感词过滤、AI 自动审核（百度 API）、多级审核策略（按用户信誉智能分流）、人工审核、审核员分配、审核日志追踪、用户信誉评分、自动通过超时内容。是**全站最复杂的模块之一**。

## 数据模型

### SensitiveWord（敏感词）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| word | CharField(100) | **是，唯一** | — | 敏感词内容 |
| is_active | BooleanField | 是 | `True` | 是否启用（可临时禁用）|
| category | CharField(50) | 否 | 空 | 敏感词分类（政治、色情等）|
| created_at | DateTimeField | 自动 | — | 添加时间 |
| updated_at | DateTimeField | 自动 | — | 更新时间 |

**自动行为**: `save()` 时自动调用 `clear_sensitive_words_cache()` 清除敏感词缓存。

### ModerationAdmin（审核管理员分配）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| target_type | CharField(20) | **是** | — | 审核内容类型: comment/topic/reply |
| admin | ForeignKey(User) | 否 | null | 审核管理员（`on_delete=SET_NULL`, `related_name='moderation_roles'`）|
| is_primary | BooleanField | 是 | `True` | 是否主要负责人（默认 True）|
| created_at / updated_at | DateTimeField | 自动 | — | 时间戳 |

> **设计说明**: 不强制 unique 约束，允许同一内容类型分配多个管理员。`get_assigned_admin()` 默认取该类型的第一个管理员作为负责人。

### ModerationReminder（审核提醒）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| target_type | CharField(20) | **是** | — | 内容类型: comment/topic/reply |
| target_id | PositiveIntegerField | **是** | — | 目标内容 ID |
| assigned_admin | ForeignKey(User) | 否 | — | 被指派的审核管理员（`on_delete=SET_NULL`, `related_name='moderation_reminders'`）|
| is_processed | BooleanField | 是 | `False` | 是否已处理 |
| created_at | DateTimeField | 自动 | — | 创建时间 |
| processed_at | DateTimeField | 否 | null | 处理时间 |

> **约束**: `unique_together = [target_type, target_id]` — 确保每个内容只生成一条提醒，避免重复通知。

### ModerationLog（审核日志）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| target_type | CharField(20) | **是** | 内容类型: comment/topic/reply |
| target_id | PositiveIntegerField | **是** | 目标内容 ID |
| action | CharField(20) | **是** | 操作: approved（通过）/ rejected（拒绝）/ reminded（提醒）|
| operator | ForeignKey(User) | 否 | 操作人（`on_delete=SET_NULL`, `related_name='moderation_logs'`，系统自动审核时为 null）|
| note | TextField | 否 | 操作备注 |
| created_at | DateTimeField | 自动 | 操作时间 |

### UserReputation（用户信誉）

> 文件: `reputation.py`

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| user | OneToOneField(User) | — | 关联用户（`related_name='reputation'`, `on_delete=CASCADE`）|
| score | IntegerField | 50 | 信誉分（0 ~ 100）|
| total_posts | IntegerField | 0 | 总发帖数 |
| approved_count | IntegerField | 0 | 审核通过数 |
| rejected_count | IntegerField | 0 | 审核拒绝数 |
| clean_days | IntegerField | 0 | 连续无违规天数 |
| last_clean_check | DateField | 自动 | 上次检查日期 |
| created_at / updated_at | DateTimeField | 自动 | 时间戳 |

**信誉等级**:

| 等级 | 分数线 | 权限变化 |
|------|--------|------|------|
| **trusted** (高信誉) | ≥ 80 | 自动发布，跳过审核 |
| **normal** (正常) | ≥ 30 | 敏感词检测，通过后发布 |
| **low** (低信誉) | < 30 | 人工审核 |

**自动化规则**:
- 连续 7 天无违规 → 奖励 5 分（可配置 `REPUTATION_WEEKLY_BONUS`）
- 发帖通过 → `approved_count++`
- 发帖拒绝 → `rejected_count++` → 可能扣分

### ReputationLog（信誉变化日志）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_reputation | ForeignKey(UserReputation) | 关联用户信誉记录（`related_name='logs'`, `on_delete=CASCADE`）|
| action | CharField(20) | 操作: score_change/level_change/bonus/penalty/manual |
| old_score | IntegerField | 操作前分数 |
| new_score | IntegerField | 操作后分数 |
| delta | IntegerField | 变化量（正增负减）|
| reason | CharField(200) | 原因说明 |
| created_at | DateTimeField | 操作时间 |

> `ordering = ['-created_at']` — 最新日志在前。

## 视图层详解

### 📊 moderation_dashboard — 审核仪表板

```python
@user_passes_test(is_moderator, login_url='accounts:login')
def is_moderator(user):
    return user.is_staff or user.is_superuser
```

**上下**: `pending_comments`, `pending_topics`, `pending_replies`

### ✅ approve_content — 页面模式：通过审核

```
@require_POST
get_content_model(content_type) → Comment/Topic/Reply
    ↓ 不支持的类型 → redirect dashboard
get_object_or_404(model, id=content_id)
    ↓
approve_instance(content, request.user, note='')
    ↓ 成功 → messages.success + redirect dashboard
    ↓ DatabaseError → messages.error + redirect dashboard
```

> **安全说明**: 仪表板中的“通过”按钮已改为 **POST 表单 + CSRF token**，避免被 GET 链接误触发或被第三方页面诱导审核。

### ❌ reject_content — 页面模式：拒绝审核

```
GET → 渲染 reject.html（拒绝理由表单）
    ↓
POST → reject_instance(content, request.user, note=review_note)
    ↓ 成功 → messages.success + redirect dashboard
    ↓ DatabaseError → messages.error + redirect dashboard
```

### 🔧 approve_content_api — API 模式：通过审核

```
@require_POST
get_content_model(content_type) → Comment/Topic/Reply
    ↓ None → 400 (MODERATION_INVALID_CONTENT_TYPE)
model.objects.filter(id=content_id).first()
    ↓ None → 404 (MODERATION_CONTENT_NOT_FOUND)
approve_instance(content, request.user, note='')
    ↓ 成功 → 200 {success: true, status: 'approved', id: xxx}
    ↓ DatabaseError → 500 (MODERATION_APPROVE_FAILED)
```

### 🔧 reject_content_api — API 模式：拒绝审核

```
@require_POST
同 approve_content_api，调用 reject_instance(content, request.user, note=review_note)
    ↓ 成功 → 200 {success: true, status: 'rejected', id: xxx}
    ↓ DatabaseError → 500 (MODERATION_REJECT_FAILED)
```

## 服务层详解 (services.py)

### 审核通过: `approve_instance(instance, operator, note)`

```
开始事务
    ↓
    instance.review_status = 'approved'
    instance.reviewed_by = operator
    instance.reviewed_at = now()
    instance.review_note = note
    instance.save()
    ↓
    创建 ModerationLog (action='approved', operator, note)
提交事务
    ↓
更新统计: 版块主题数、回复数
    ↓
异常: DatabaseError → 记录日志 → 抛出
```

### 审核拒绝: `reject_instance(instance, operator, note)`

```
开始事务
    ↓
    instance.review_status = 'rejected'
    instance.reviewed_by = operator
    instance.reviewed_at = now()
    instance.review_note = note  # 拒绝理由
    instance.save()
    ↓
    创建 ModerationLog (action='rejected', operator, note)
提交事务
    ↓
更新统计
    ↓
异常: DatabaseError → 记录日志 → 抛出
```

### 系统自动通过: `auto_approve_instance(instance, note)`

与 `approve_instance` 类似，但：
- `reviewed_by = None`（系统操作，非人工）
- `ModerationLog.operator = None`

### AI 智能审核: `smart_moderate_instance(instance, content)`

```
获取内容（标题 + 正文）
    ↓
敏感词检测（本地词库）
    ↓
百度 AI 审核（get_moderation_summary）
    ↓
    ├── approved → auto_approve_instance() → 直接通过
    ├── rejected → 自动拒绝 + 记录违规原因
    └── pending → 标记待审核 + 进入人工队列
```

### 批量 AI 审核: `ai_batch_moderate(model_class, batch_size=100)`

```
获取 pending 状态的内容（最多 batch_size 条）
    ↓
逐条调用 smart_moderate_instance()
    ↓
统计: {approved: N, rejected: N, pending: N, error: N}
```

### 辅助函数

| 函数 | 说明 |
|------|------|
| `get_target_type(instance)` | 根据实例类型返回 'comment'/'topic'/'reply' |
| `update_related_counts(instance, target_type)` | 更新关联统计（版块计数、回复计数）|
| `get_assigned_admin(target_type)` | 获取指定类型的审核管理员 |
| `create_moderation_reminder(target_type, target_id)` | 创建审核提醒 + 记录日志 |

## 多级审核策略引擎 (strategy.py) 🆕

> 这是**最核心的审核决策逻辑**，根据用户信誉智能分流审核。

### ModerationStrategy 类

```python
def __init__(self):
    self.moderation_service = get_moderation_service()
    self.mode = getattr(settings, 'MODERATION_MODE', 'hybrid')
    self.ai_threshold = getattr(settings, 'MODERATION_AI_THRESHOLD', 0.7)
```

### 审核策略表

| 用户信誉等级 | 内容检查 | 审核结果 | 说明 |
|-------------|----------|----------|------|
| **trusted** (≥80) | 敏感词检查 | 无敏感词 → 自动通过 | 快速策略 |
| | | 有敏感词 → 待人工审核 | 降级到人工 |
| **normal** (≥30) | 敏感词 + AI 双检 | 敏感词 → 人工审核 | 标准策略 |
| | | AI 通过 → 自动发布 | AI 确认安全 |
| | | AI 拒绝 → 自动拒绝 | AI 确认违规 |
| | | AI 不确定 → 人工审核 | 保守处理 |
| **low** (<30) | 直接人工审核 | 总是待审核 | 严格策略 |

### get_review_strategy(user, content)

返回 `(策略, 详情)` 元组。

**策略类型**:

| 策略 | 触发条件 |
|------|----------|
| `auto_publish` | trusted 用户无敏感词 / AI 确认通过 |
| `sensitive_review` | 高信誉用户命中敏感词 / 普通用户命中敏感词 |
| `ai_rejected` | AI 确认违规 |
| `ai_pending` | AI 不确定 |
| `manual_review` | low 信誉用户 / AI 出错 / AI pending |

### moderate_content(user, content, content_type='text')

返回完整的审核结果：
```python
{
    'status': 'approved' / 'rejected' / 'pending',
    'strategy': 'auto_publish' / 'manual_review' / ...,
    'details': {
        'user_level': 'trusted' / 'normal' / 'low',
        'has_sensitive': True/False,
        'sensitive_words': [...],
        'ai_result': 'approved' / 'rejected' / 'pending',
        'ai_details': {...},
    },
    'message': '自动审核通过',
}
```

## 百度 AI 审核服务 (ai_service.py) 🆕

### BaiduModerationService 类

| 方法 | 说明 |
|------|------|
| `moderate_text(content)` | 文本审核 → ('approved'/'rejected'/'pending'/'error', 违规详情) |
| `moderate_image(image_path)` | 图片审核 → 读取文件 → 调用百度 API |

| 懒加载 client | 首次调用时初始化 `AipContentCensor` |
| 错误处理 | service_unavailable_payload() 统一错误返回 |

### 判定标准

| conclusionType | 含义 | 返回结果 |
|----------------|------|----------|
| 1 | 合规 | approved |
| 2 | 疑似 | pending |
| 3 | 不合规 | rejected |
| 其他 | 未知结论 | pending |

### MockModerationService 类

开发/测试环境使用的模拟审核服务：
- 内置模拟违规词: `['测试违规词', '广告测试', '敏感测试']`
- 命中 → rejected, 否则 approved
- 图片 → 始终 approved

### 全局函数: `get_moderation_service()`

```python
@lru_cache(maxsize=1)  # 进程级别缓存
def get_moderation_service():
    if settings.BAIDU_MODERATION_ENABLED:
        return BaiduModerationService()
    return MockModerationService()
```

```

    ↓

    ↓
get_moderation_strategy().get_review_strategy(user, content)
    ↓
┌─────────────────────────────────────────────────────┐
│              多级审核策略引擎                          │
│                                                      │
│  1. 获取用户信誉等级                                  │
│     ↓                                                │
│     ├── trusted → 快速敏感词检查                      │
│     │   ├── 无敏感词 → auto_publish                  │
│     │   └── 有敏感词 → sensitive_review              │
│     │                                                │
│     ├── low → manual_review (强制人工)               │
│     │                                                │
│     └── normal → 敏感词 + AI 双重检测                │
│         ├── 敏感词命中 → sensitive_review             │
│         ├── AI approved → auto_publish               │
│         ├── AI rejected → ai_rejected                │
│         └── AI pending → manual_review               │
│                                                      │
│  2. 根据策略执行                                      │
│     ├── auto_publish → auto_approve                   │
│     ├── ai_rejected → reject_instance                 │
│     ├── sensitive_review → 标记 pending               │
│     └── manual_review → 标记 pending + 提醒           │
│                                                      │
│  3. 记录日志                                          │
│     → ModerationLog.create()                         │
│     → 更新统计信息                                    │
└─────────────────────────────────────────────────────┘
```

## Celery 异步任务 (tasks.py) 🆕

> **所有任务支持自动重试**: `max_retries=3, default_retry_delay=60s`

### 任务 1: async_moderate_text — 异步文本审核

```
获取用户 (user_id) → 可选
    ↓
get_moderation_strategy().moderate_content(user, content)
    ↓
更新对象 review_status + review_note
    ↓
创建 ModerationLog
    ↓
更新用户信誉:
    ├── approved → +1 分 (REPUTATION_APPROVE_BONUS)
    └── rejected → -5 分 (REPUTATION_REJECT_PENALTY)
    ↓
如果 pending → create_moderation_reminder()
```

**参数**: `content_type`, `content_id`, `content`, `user_id`

### 任 2: async_moderate_image — 异步图片审核

```
获取用户 (user_id)
    ↓
strategy.moderate_image(user, image_path, image_url)
    ↓
更新状态 + 创建 ModerationLog
```

**参数**: `content_type`, `content_id`, `image_path`, `image_url`, `user_id`

## 错误处理

| 场景 | 行为 |
|------|------|
| 审核异常 | DatabaseError → 记录日志 → 抛出 (不静默) |
| AI 服务不可用 | 标记为 pending，进入人工审核队列 |
| 敏感词检查 | 命中敏感词 → pending（即使是高信誉用户）|
| 缓存未命中 | 重新加载敏感词列表 |
| 百度 API 超时 | 返回 error，记录日志 |
