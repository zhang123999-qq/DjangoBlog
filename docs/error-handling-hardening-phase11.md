# 异常处理分层收敛（阶段 11：拒绝原因弹层）

覆盖文件：
- `moderation/templates/moderation/dashboard.html`

## 目标

将拒绝审核时的 `window.prompt` 升级为内嵌弹层，提升交互质量和输入可控性。

## 本次实现

1. 审核按钮统一接入 JS 行为（comment/topic/reply 全覆盖）
2. 新增拒绝原因弹层（纯前端，无额外依赖）
3. 弹层能力：
   - 字数限制（200）
   - 实时计数
   - 常用模板快速填充（广告/敏感内容/质量低）
   - 取消 / 确认
4. 取消时不发请求，恢复按钮状态
5. 提交后仍沿用错误码映射和 Toast 反馈

## 收益

- 操作更专业，不再出现系统 prompt
- 拒绝原因质量更高（模板 + 字数约束）
- 保持与 Phase 9/10 的交互一致性
