# 模块 02 — 博客核心

> 路径: `apps/blog/` | URL前缀: `/blog/` | 项目核心功能模块

## 功能概览

博客的内容管理中：文章创建、编辑、删除、发布、归档，分类和标签管理，浏览统计（Redis 计数），评论系统，以及 Celery 异步任务（浏览量同步、热门章、草稿清理）。

## 数据模型

###  Category（分类）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | CharField(100) | **是，唯一** | 分类名称 |
| slug | SlugField(100) | 是，唯一 | URL 别名（自动生成，支持中文）|
| created_at | DateTimeField | 自动 | 创建时间 |
| updated_at | DateTimeField | 自动 | 更新时间 |

**自动行为**: `save()` 时如果 slug 为空，自动调用 `generate_slug(name)` 生成（支持中文转拼音）。

###  Tag（标签）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | CharField(50) | **是，唯一** | 标签名称 |
| slug | SlugField(50) | 是，唯一 | URL 别名（自动生成）|
| created_at | DateTimeField | 自动 | 创建时间 |
| updated_at | DateTimeField | 自动 | 更新时间 |

### 📝 Post（文章）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | CharField(200) | **是** | 文章标题 |
| slug | SlugField(200) | 是，唯一 | URL 别名（自动生成，冲突时加数字后缀）|
| summary | TextField(500) | 否 | 摘要（SEO 和列表页展示用）|
| content | TextField | **是** | 正文内容（使用 TinyMCE 编辑器）|
| author | ForeignKey(User) | **是** | 作者（`on_delete=CASCADE`，`related_name='posts'`）|
| category | ForeignKey(Category) | 否 | 分类（`on_delete=SET_NULL`，`null=True`，`related_name='posts'`）|
| tags | ManyToManyField(Tag) | 否 | 标签（`blank=True`，`related_name='posts'`）|
| status | CharField(20) | 是 | 状态：draft(草稿)/published(已发布)/archived(已归档)，默认 draft |
| allow_comments | BooleanField | 是 | 是否允许评论，默认 True |
| views_count | PositiveIntegerField | 是 | 浏览量（Redis 计数 + 异步同步到数据库，认 0）|
| published_at | DateTimeField | 否 | 发布时间（首次从 draft→published 时自动设置）|
| created_at | DateTimeField | 自动 | 创建时间 |
| updated_at | DateTimeField | 自动 | 更新时间 |

## 视图层详解

### 📋 PostListView — 文章列表

**功能**: 展示已发布文章列表，支持分类和标签筛选。

**查询优化**:
```
filter(status='published')
    .select_related('author', 'category')     -- 预加载关联对象
    .prefetch_related(Prefetch('tags', ...))   -- 预加载标签
    .only('title', 'slug', 'summary', ...)     -- 只查询需要的字段
```

**上下文数据**: 文章列表 (posts)、分类列表、标签列表、当前分类或标签。

### 📖 PostDetailView — 文章详情

**功能**: 展示文章完整内容，包括作者信息、分类、标签、评论列表和评论表单。

**查询优化**:
```
select_related('author', 'category')
.prefetch_related(
    tags,
    Prefetch('comments', queryset=Comment.filter(review_status='approved').select_related('user'))
)
```

**行为**:
- 获取文章对象
- 如果是已发布文章 → 调用 `increase_views()` 增加浏览数
- 预加载已审核评论
- 填充 CommentForm（登录用户隐藏姓名邮箱字段）

### ✍️ PostCreateView — 创建文章

**权限**: LoginRequiredMixin（必须登录）
**限流**: 每小时最多创建 5 篇（`@rate_limit_by_user('post_create', rate='5/h')`）
**表单**: PostForm（TinyMCE 富文本编辑器）
**自动设置**: `form.instance.author = request.user`

### ✏️ PostUpdateView — 编辑文章

**权限**: LoginRequiredMixin + UserPassesTestMixin
- `test_func()`: 只有文章作者或管理员 (is_staff) 可以编辑
- `slug_field='slug'`: 通过 slug 查找文章

### 🗑️ PostDeleteView — 删除文章

**权限**: 同 PostUpdateView
**行为**: 删除后跳转到文章列表，显示成功消息

###  Comment — 评论发表

**限流**: 每分钟最多 10 条评论（`@rate_limit('comment', rate='10/m')`）

```
POST 请求:
    ↓ 获取文章（仅 published 状态）
    ↓ 验证 CommentForm
    ↓ 创建 Comment 对象
    ↓ 登录用户 → 关联 user
    ↓ 游客 → 记录 name, email
    ↓ 记录 IP 地址 + User Agent
    ↓ 保存为 pending 状态
    ↓ 调用 smart_moderate_instance() → AI 审核
    ↓ approved → "评论已发布"
    ↓ rejected → "评论未通过审核：原因"
    ↓ pending → "评论已提交，等待审核"
    ↓ 重定向回文章
```

### 👍 like_comment_view — 点赞评论

**权限:** 必须登录
**限流:** 每分钟 30 次点赞
**行为:**
1. 只能对已审核通过的评论点赞
2. 未点赞 → 点赞 (get_or_create)
3. 已点赞 → 取消点赞 (delete)
4. 更新评论的 like_count
5. 返回 JSON: `{success, liked, like_count, message}`

### 📝 my_posts — 我的文章

**功能:** 显示当前用户发表的所有文章（所有状态：draft/published/archived），分页 20 条。

### 📋 post_draft_list — 我的草稿

**功能:** 只显示当前用户写的草稿（status='draft'），分页 20 条。

## Celery 异步任务 (`tasks.py`)

### 📊 sync_views_to_db — 浏览量同步

**执行频率:** 每 5 分钟一次

```
扫描 Redis keys: post:views:*
    ↓ 提取 post_id 和 views_count
    ↓ 逐条执行: Post.objects.filter(pk=post_id).update(views_count=F('views_count') + views)
    ↓ 删除已同步的 Redis keys
    ↓ 统计: 同步文章数、总浏览量
```

**容错:** 更新失败的文章重新写入 Redis 缓存，下次任务重试。

### 🔥 update_hot_posts — 热门文章统计

**执行频率:** 每 1 小时一次

| 缓存键 | 范围 | 数量 | TTL |
|--------|------|------|-----|
| `blog:hot_posts:week` | 最近 7 天 | Top 10 | 3600s |
| `blog:hot_posts:month` | 最近 30 天 | Top 20 | 3600s |

### 🧹 cleanup_old_drafts — 清理旧草稿

**执行频率:** 每周一次

**行为:** 删除超过 90 天未更新的草稿（status='draft', updated_at < 90 天前）。

## 缓存系统

### 缓存策略

| 缓存键 | 内容 | TTL | 清除时机 |
|--------|------|-----|----------|
| `blog_categories_tags` | 分类列表 + 标签列表（带文章数）| 5 分钟 | 文章增删改后自动清除 |
| `post:views:{id}` | 浏览量计数 | 3600 秒 | sync_views_to_db 同步后清除 |
| `blog:hot_posts:week` | 本周热门章 ID 列表 | 3600 秒 | update_hot_posts 任务更新 |
| `blog:hot_posts:month` | 本月热门文章 ID 列表 | 3600 秒 | update_hot_posts 任务更新 |

### `get_categories_and_tags()` 函数

被多个视图调用，内部实现缓存：
1. 先检查缓存（5 分钟有效）
2. 缓存命中则直接返回
3. 缓存未命中 → 查询数据库（只查询 name + slug 字段 + annotate 文章数）→ 写入缓存

## 浏览计数流程（Redis）

```
用户访问文章详情页
    ↓ PostDetailView.get_object()
    ↓ post.increase_views() [models.py]
    ↓
    ┌── 尝试 Redis INCR (cache.incr):
    │   ├── key 不存在 → cache.set(key, 1, 3600)
    │   ├── key 存在 → cache.incr(key)
    │   └── 更新内存值（仅本次请求的乐观显示）
    │
    └── 失败时降级:
        └── Post.objects.filter(pk=pk).update(views_count=F() + 1)

每 5 分钟 → Celery sync_views_to_db → Redis 值批量更新到数据库
```

## 文章生命周期

```
创建 → draft（草稿，仅作者可见）
    ↓ 发布
published（已发布，所有人可见）
    ↓ 归档
archived（已归档，可通过列表页访问）
    ↓ 编辑
  └─ 作者/管理员可编辑任何状态的文章
    ↓ 删除
  └─ 作者/管理员可删除任何状态的文章
```
