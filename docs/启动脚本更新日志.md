## 使用方法

### 检查环境
```bash
python run.py --check
```

### 开发环境安装
```bash
python run.py
# 或
python run.py --dev
```

### 生产环境安装
```bash
python run.py --prod
```

### 非交互模式
```bash
python run.py --no-input
```

## 命令行参数

| 参数 | 说明 |
|------|------|
| `--prod` | 生产环境安装 |
| `--dev` | 开发环境安装（默认） |
| `--check` | 仅检查系统环境 |
| `--no-input` | 非交互模式 |

## 安装步骤

1. 检查系统环境
   - Python 版本检测
   - pip/uv 检测
   - 项目文件完整性检查

2. 创建虚拟环境
   - 检测现有虚拟环境
   - 支持删除重建

3. 安装依赖
   - 升级 pip
   - 安装项目依赖
   - 失败时安装核心包

4. 配置环境变量
   - 创建 .env 文件
   - 自动生成 SECRET_KEY

5. 数据库迁移
   - 生成迁移文件
   - 执行迁移

6. 收集静态文件
   - 自动检测是否需要

7. 创建管理员账户
   - 检测现有管理员
   - 交互式创建

8. 安装 Gunicorn（生产环境）
   - 安装 Gunicorn
   - 生成启动脚本

## 日志文件

安装日志保存在 `logs/install_YYYYMMDD_HHMMSS.log`

## 生成文件

- `.env` - 环境变量配置
- `.venv/` - 虚拟环境目录
- `logs/` - 日志目录
- `start_server.sh` - Linux/Mac 启动脚本
- `start_server.bat` - Windows 启动脚本

## 常见问题处理

### 终端乱码
- 自动检测并设置 UTF-8 编码
- Windows 下自动执行 `chcp 65001`

### Python 版本不满足
- 显示当前版本和需要版本
- 提示安装更高版本

### pip 安装失败
- 尝试使用 uv
- 降级安装核心包

### 虚拟环境已存在
- 提示删除或使用现有环境
- 自动检测虚拟环境完整性

### 端口被占用
- start.py 自动检测端口占用
- 提示使用其他端口
