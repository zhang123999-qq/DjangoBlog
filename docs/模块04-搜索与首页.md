## 核心功能详解

### 🏠 首页 (`home_view`)

**功能**: 展示全站最精华内容，整合三大板块（博客文章、论坛主题、工具箱）的最新热门内容，并显示统计数据。

**数据加载**:

| 数据块 | 查询 | 数量 | 优化方式 |
|--------|------|------|----------|
| **最新文章** | `Post.objects.filter(status='published')` → `.select_related('author', 'category')` → `.only(...)` → `.order_by('-created_at')[:6]` | 最 6 篇 | `select_related` 预加载作者和分类，`.only()` 只查询展示所需字段 |
| **热门帖子** | `Topic.objects.filter(review_status='approved')` → `.select_related('board', 'author')` → `.order_by('-reply_count')[:5]` | 最 5 个 | `select_related` 预加载版块和作者，按回复数排序 |
| **热门工具** | `tool_registry.get_all_tools()[:8]` | 最多 8 个 | Registry 缓存（1 小时）|
| **数据** | 6 个 count/aggregate 查询 | 统计值 | 无特殊优化 |

**上下文变量**:

| 变量 | 说明 |
|------|------|
| `latest_posts` | 最新 6 篇已发布文章 |
| `hot_topics` | 热门 5 个论坛主题 |
| `popular_tools` | 最 8 个工具 |
| `post_count` | 已发布文章总数 |
| `topic_count` | 已审核论坛主题总数 |
| `comment_count` | 已审核评论总数 |
| `user_count` | 用户总数 |
| `tool_count` | 可用工具总数 |
| `view_count` | 全站文章总浏览量 |

**缓存策略**:
```
缓存 Key: core:home:v1
TTL: 60 秒
内容: 完整的 context 字典（所有数据直接渲染）
流程: 缓存命中 → 直接渲染 home.html
      缓存未命中 → 查询数据 → 写入缓存 → 渲染
```

> **设计考量**: 首页是最高频访问的页面，60 秒缓存可以大幅降低数据库压。对于文章发布等实时更新场景，用户可以接受 60 秒的延迟。

### 🔍 全局搜索 (`search_view`)

**功能**: 跨模块搜索文章标题/正文/摘要和论坛主题标题/正文。

**搜索范围**:

| 模块 | 搜索字段 | 过滤条件 | 返回数量 |
|------|----------|----------|----------|
| **文章** | `title__icontains`, `content__icontains`, `summary__icontains` | `status='published'` | 最多 20 篇 |
| **论坛主题** | `title__icontains`, `content__icontains` | `review_status='approved'` | 最多 20 篇 |

**使用方式**:
```
GET /search/?q=搜索关键词
```

**上下文变量**:

| 变量 | 说明 |
|------|------|
| `query` | 用户输入的搜索词 |
| `results['posts']` | 匹配的文章列表 |
| `results['topics']` | 匹配的论坛主题列表 |

**限制**:
- 空搜索词（`q=''`）→ 不执行查询，返回空结果
- 使用 `icontains` 模糊匹配（大小写不敏感）
- 使用 `distinct()` 去重（多字段匹配可能产生重复）
- 最多返回 20 条结果（无分页）

**模板**: `templates/search/results.html`

## 健康检查端点

### 🏥 healthz_view — 增强健康检查

**用途**: Kubernetes/Docker 兼容的健康检查端点，用于容器探针检测。

**检查项**:

| 检查项 | 实现方式 | 失败条件 |
|--------|----------|----------|
| **数据库** | `SELECT 1` 查询 | `OperationalError` / `DatabaseError` / `OSError` |
| **缓存 (Redis)** | `set` → `get` → `delete` 验证 | 值不匹配 或 连接/超时/OSError |

**响应格式**:

```json
{
    "status": "healthy",
    "checks": {
        "database": true,
        "cache": true
    },
    "duration_ms": 1.23,
    "version": "2.3.2"
}
```

| 状态 | HTTP 码 | 条件 |
|------|---------|------|
| `healthy` | **200** | 所有检查项通过 |
| `unhealthy` | **503** | 任一检查项败 |

### 📊 readiness_view — 就绪检查

**用途**: 判断服务是否准备好接收请求（K8s Readiness Probe）。

**检查项**: 同 healthz（数据库 + 缓存）

**响应**:

```json
{
    "ready": true,
    "checks": {
        "database": true,
        "cache": true
    }
}
```

### 💓 liveness_view — 存活检查

**用途**: K8s Liveness Probe — 检查进程是否存活（最简单的检查）。

**响应**:

```json
{
    "alive": true
}
```

只返回 `{"alive": true}`，不检查任何外部依赖。如果进程挂了，K8s 无法收到响应，就会重启容器。

## 系统设置 (`settings_view`)

**权限**: 必须登录 + `is_staff` + `DEBUG=True`（生产环境禁用）

**环境变量文件**: `F:\DjangoBlog\.env`

### `read_env_config()` — 读取 .env

```
逐行读取 .env 文件
    ↓ 跳过空行和注释行（以 # 开头）
    ↓ 按 = 分割键值对
    ↓ 返回 dict
```

### `save_env_config(config)` — 保存 .env

```
写入 .env 文件（覆盖模式）
    ↓ 写入 # Django 配置 注释
    ↓ 写入生成时间（固定 2026-03-20）
    ↓ 逐行写入 key=value
```

### 可配置的项

| 配置项 | 说明 |
|--------|------|
| `SITE_NAME` | 站点名称 |
| `SITE_DESCRIPTION` | 站点描述 |
| `SITE_URL` | 站点 URL |
| `ALLOWED_HOSTS` | 允许的主机（`allow_lan` 开 → `*`，否则 → `localhost,127.0.0.1`）|
| `DEBUG` | 调式模式 |

> **⚠️ 安全警告**: 该页面在生产环境被 `HttpResponseForbidden` 禁用，避免了误操作修改线上配。

## 性能优化

| 优化项 | 说明 |
|--------|------|
| 首页整体缓存 | 60 秒缓存，命中时跳过所有查询 |
| `select_related` | 预加载作者和分类/版块，避免 N+1 |
| `.only()` | 只查询模板实际使用的字段，减少数据传输 |
| 聚合查询 | `Sum('views_count')` 替代 Python 计算 |
| Registry 缓存 | `get_all_tools()` 使用 Django 缓存（1 小时）|

## 安全检查

| # | 安全措施 | 实现 |
|---|----------|------|
| 1 | 生产环境禁设置 | `settings.DEBUG` 为 False 时返回 `HttpResponseForbidden` |
| 2 | 设置权限 | 必须 `@login_required` + `@user_passes_test(lambda u: u.is_staff)` |

| 4 | 搜索词清洗 | `request.GET.get('q', '').strip()` 去除首尾空格 |
| 5 | 数据库检查 | `SELECT 1` 轻量级检查，不执行复杂查询 |
| 6 | 缓存检查 | set + get + delete，10 秒 TTL，不污染缓存 |

## 错误处理

| 场景 | 行为 |
|------|------|
| 首页缓存未命 + 查询失败 | 异常会传播（无 try/except），Django 500 页面 |
| 数据库不可用 | healthz 返回 503 `{"status": "unhealthy", "checks": {"database": false}}` |
| Redis 不可用 | healthz 返回 503 `{"status": "unhealthy", "checks": {"cache": false}}` |
| 空搜索词 | 返回空结果页面（不报错）|
| .env 文件不存在 | `get_env_config()` 返回空 dict |
| 非 staff 访问设置页 | 重定向到登录页（`@login_required` + `@user_passes_test`）|
| 生产环境访问设置页 | `403 Forbidden` |
