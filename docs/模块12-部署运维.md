### 启动顺序（依赖链 + 健康检查）

```
Phase 1: 基础设施
    ├── db (MySQL) ← healthcheck: mysqladmin ping (10s 间隔, 5 次重试)
    └── redis      ← healthcheck: redis-cli ping  (10s 间隔, 5 次重试)

Phase 2: 数据库迁移（一次性）
    └── migrate (depends_on: db healthy + redis healthy)
        → 执行 python manage.py migrate --noinput
        → restart: "no"（完成后不重启）

Phase 3: 应用服务
    ├── web (depends_on: migrate service_completed_successfully)
    │   → healthcheck: HTTP GET /healthz/ (30s 间隔)
    │
    ├── celery_worker (depends_on: db healthy + redis healthy + migrate success)
    │
    └── celery_beat (depends_on: redis healthy + migrate success)

Phase 4: 反向代理
    └── nginx (depends_on: web service_healthy)
        → healthcheck: wget --spider -q http://localhost/healthz/
```

### 服务详细配置

#### MySQL 8.0

```yaml
command: >
  --character-set-server=utf8mb4
  --collation-server=utf8mb4_unicode_ci
  --default-authentication-plugin=caching_sha2_password
  --max-connections=200
  --innodb-buffer-pool-size=256M
```

- 字符集: utf8mb4（支持 emoji）
- 最大连接数: **200**
- InnoDB Buffer Pool: **256MB**
- 认证插件: caching_sha2_password

#### Redis 7

```yaml
command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru --save 60 1
```

- 最大内存: **256MB**
- 淘汰策略: **allkeys-lru**（LRU 淘汰所有 key）
- 持久化: 60 秒内有 1 次变更则保存

#### Web (Django + Gunicorn)

```yaml
command: [
  "gunicorn", "config.wsgi:application",
  "--bind", "0.0.0.0:8000",
  "--workers", "4",
  "--worker-class", "gevent",
  "--worker-connections", "1000",
  "--timeout", "120",
  "--access-logfile", "-",
  "--error-logfile", "-"
]
```

- **唯一 build 服务**：其他服务复用此 image
- Worker: 4 进程 × gevent（异步）× 1000 连接/进程 = **4000 并发**
- 超时: 120s
- 日志: 标准输出（Docker 日志驱动）

#### Celery Worker

```yaml
command: [
  "celery", "-A", "config", "worker",
  "--loglevel=info",
  "--concurrency=4",
  "--pool=prefork",
  "--max-tasks-per-child=1000",
  "--queues=celery,low_priority"
]
```

- 并发: **4 进程**（prefork 池）
- 每个 Worker 最大任务数: **1000**（自动重启防内存泄漏）
- 队列: **celery**（默认）+ **low_priority**（低优先级）

#### Celery Beat

```yaml
command: ["celery", "-A", "config", "beat", "--loglevel=info"]
```

- 仅调度，不执行任务
- 依赖: Redis + migrate 完成

#### Nginx

```yaml
volumes:
  - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro          # 配置
  - static_volume:/app/staticfiles:ro                        # 静态文件
  - media_volume:/app/media:ro                               # 媒体文件
  - ./nginx/ssl:/etc/nginx/ssl:ro                            # SSL 证书
```

- 挂载 4 个卷（2 个 SSL 证书目录）
- depends_on: web 健康检查通过

## 二、Dockerfile（`deploy/Dockerfile`）

> **双阶段构建**（builder + final），极大减小最终镜像体积。

### 阶段 1: Builder（编译阶段）

```dockerfile
FROM docker.1ms.run/library/python:3.13-slim AS builder
```

| 步骤 | 说明 |
|------|------|
| **基础镜像** | Python 3.13-slim（通过 `docker.1ms.run` 加速）|
| **apt 源切换** | 删除 debian.sources → 写入清华 trixie 源 |
| **安装编译依赖** | gcc, pkg-config, libffi-dev, libjpeg-dev, libpng-dev, zlib1g-dev, libssl-dev |
| **pip 源切换** | 清华 PyPI 镜像（pypi.tuna.tsinghua.edu.cn）|
| **安装 Python 依赖** | `pip install -r requirements/production.txt`（--no-cache-dir, --timeout 120, --retries 5）|

### 阶段 2: Final（运行阶段）

```dockerfile
FROM docker.1ms.run/library/python:3.13-slim
```

| 步骤 | 说明 |
|------|------|
| **基础镜像** | Python 3.13-slim（同加速源）|
| **apt 源切换** | 同 builder（清华 trixie 源）|
| **安装运行时依赖** | libjpeg62-turbo, libpng16-16（仅需运行时库）|
| **复制 Python 包** | `COPY --from=builder site-packages` |
| **复制可执行文件** | `COPY --from=builder /usr/local/bin` |
| **创建目录** | staticfiles / media / logs（权限 777）|
| **复制项目代码** | manage.py + apps/ + config/ + moderation/ + templates/ + static/ + requirements/ |
| **文件归属** | `--chown=root:root`（安全最佳实践）|
| **EXPOSE** | 8000 |

## 五、一键部署脚本（`auto-deploy.sh`）

> 11.7KB，**7 种运行模式** + 部署时 **12 个步骤** + Trap 优雅退出 + 国内镜像自动配置 + 交互式管理员创建。

### ⚠️ 重要说明

此脚本 **需要 root 权限** 运行：`sudo bash deploy/auto-deploy.sh`

### Trap 信号处理（优雅退出）

```bash
trap cleanup INT TERM
```

Ctrl+C 或 SIGTERM 中断 → 输出黄色提示："脚本中断，如需要重新启动：Docker restart 或者重启服务器" → 退出码 130

### 7 种运行模式

| 模式 | 命令 | 功能 |
|------|------|------|
| **deploy**（默认） | `bash auto-deploy.sh` 或 `bash auto-deploy.sh deploy` | 完整部署流程（12 步骤）|
| **status** | `bash auto-deploy.sh status` | 查看服务状态 + 镜像列表 + 磁盘使用 |
| **update** | `bash auto-deploy.sh update` | Git pull + 重新构建 + 迁移 + 重启 |
| **stop** | `bash auto-deploy.sh stop` | dc down 停止所有服务 |
| **restart** | `bash auto-deploy.sh restart` | dc restart + 状态报告 |

**dc 函数**（统一 docker compose 调用）:
```bash
dc() { docker compose --env-file "$ENV_FILE" -f "$COMPOSE_FILE" "$@"; }
```

### 部署模式 12 个步骤

#### 步骤 1: 自动创建 .env（如不存在）

```bash
SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_urlsafe(50))")
```
