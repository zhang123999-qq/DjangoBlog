# 模块 05 — 社区论坛

> 路径: `apps/forum/` | URL前缀: `/forum/` | 独立应用模块

---

## 功能概览

社区论坛系统：多版块管理、话题发布、回复讨论、浏览统计、置顶/锁定、智能审核（敏感词 + AI）、回复点赞。完整的内容社区体验。

## 数据模型

### Board（版块）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| name | CharField(100) | **是，唯一** | — | 版块名称 |
| slug | SlugField(100) | 是，唯一 | 空 | URL 别名（自动生成，支持中文拼音）|
| description | TextField | 否 | 空 | 版块描述 |
| topic_count | PositiveIntegerField | 是 | 0 | 主题数（自动更新，只统计 approved）|
| reply_count | PositiveIntegerField | 是 | 0 | 回复数（自动更新，只统计 approved 且 is_deleted=False）|
| last_post_at | DateTimeField | 否 | null | 最后发帖时间 |
| created_at | DateTimeField | 自动 | — | 创建时间 |
| updated_at | DateTimeField | 自动 | — | 更新时间 |

**方法**:

| 方法 | 说明 |
|------|------|
| `save()` | 自动生成 slug（调用 `generate_slug(self.name)`）|
| `update_counts()` | 聚合查询更新：topic_count、reply_count、last_post_at。`topic_count` = 已审核主题数；`reply_count` = 已审核主题的 reply_count 聚合和；`last_post_at` = 最新回复时间 |
| `get_absolute_url()` | 返回版块主题列表页 URL |

### Topic（主题）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| board | ForeignKey(Board) | **是** | — | 所属版块（`on_delete=CASCADE`，`related_name='topics'`，`related_name='board'`） |
| author | ForeignKey(User) | **是** | — | 作者（`on_delete=CASCADE`，`related_name='topics'`）|
| title | CharField(200) | **是** | — | 主题标题 |
| content | TextField | **是** | — | 内容 |
| views_count | PositiveIntegerField | 是 | 0 | 浏览量 |
| reply_count | PositiveIntegerField | 是 | 0 | 回复数（自动更新）|
| is_pinned | BooleanField | 是 | False | 是否置顶 |
| is_locked | BooleanField | 是 | False | 是否锁定（禁止回复）|
| review_status | CharField(20) | 是 | `pending` | 审核状态: pending/approved/rejected |
| reviewed_by | ForeignKey(User) | 否 | null | 审核人（`on_delete=SET_NULL`，`related_name='reviewed_topics'`）|
| reviewed_at | DateTimeField | 否 | null | 审核时间 |
| review_note | TextField | 否 | 空 | 审核备注（拒绝理由）|
| last_reply_at | DateTimeField | 否 | null | 最后回复时间（更新回复时自动设置）|
| created_at | DateTimeField | 自动 | — | 创建时间 |
| updated_at | DateTimeField | 自动 | — | 更新时间 |

**索引**:

| 索引类型 | 字段 | 用途 |
|----------|------|------|
| 联合索引 | `board` + `last_reply_at`(降序) | 版块内主题按最后回复排序 |
| 常规索引 | `author` | 按作者筛选 |
| 常规索引 | `review_status` | 按审核状态筛选 |
| 常规索引 | `is_pinned` | 置顶优先排序 |
| 常规索引 | `created_at`(降序) | 按创建时间排序 |

**排序规则**: `ordering = ['-is_pinned', '-last_reply_at', '-created_at']` → 置顶帖 → 最后回复时间 → 创建时间

**方法**:

| 方法 | 说明 |
|------|------|
| `increase_views()` | 浏览量 F() 更新 + 内存值 +1（简单版，无 Redis 缓存）|
| `update_reply_count()` | 统计已审核且未删除的回复数 → 更新 `reply_count` 和 `last_reply_at` → 触发 `board.update_counts()` |
| `get_absolute_url()` | 返回主题详情页 URL：`/forum/board/{slug}/topic/{id}/` |
| `is_pending` (property) | `review_status == 'pending'` |
| `is_approved` (property) | `review_status == 'approved'` |
| `is_rejected` (property) | `review_status == 'rejected'` |

### Reply（回复）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| topic | ForeignKey(Topic) | **是** | — | 回覆的主题（`on_delete=CASCADE`，`related_name='replies'`）|
| author | ForeignKey(User) | **是** | — | 回复者（`on_delete=CASCADE`，`related_name='replies'`）|
| content | TextField | **是** | — | 回复内容 |
| like_count | PositiveIntegerField | 是 | 0 | 点赞数 |
| is_deleted | BooleanField | 是 | False | 软删除标记（不真正删除数据）|
| review_status | CharField(20) | 是 | `pending` | 审核状态: pending/approved/rejected |
| reviewed_by | ForeignKey(User) | 否 | null | 审核人（`on_delete=SET_NULL`，`related_name='reviewed_replies'`）|
| reviewed_at | DateTimeField | 否 | null | 审核时间 |
| review_note | TextField | 否 | 空 | 审核备注 |
| created_at | DateTimeField | 自动 | — | 创建时间 |
| updated_at | DateTimeField | 自动 | — | 更新时间 |

**索引**:

| 索引类型 | 字段 | 用途 |
|----------|------|------|
| 联合索引 | `topic` + `created_at`(降序) | 主题内回复按时间排序 |
| 常规索引 | `author` | 按作者筛选 |
| 常规索引 | `review_status` | 按审核状态筛选 |
| 常规索引 | `created_at`(降序) | 按时间排序 |

**排序规则**: `ordering = ['created_at']` → 按时间正序

**方法**:

| 方法 | 说明 |
|------|------|
| `update_like_count()` | 从 `likes` 关系计算实际点赞数，用 `update()` F() 避免竞态条件，然后同步到内存 |
| `is_pending/is_approved/is_rejected` (properties) | 审核状态判断 |

### ReplyLike（回复点赞）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user | ForeignKey(User) | **是** | 点赞用户（`on_delete=CASCADE`，`related_name='reply_likes'`）|
| reply | ForeignKey(Reply) | **是** | 被点赞的回复（`related_name='likes'`）|
| created_at | DateTimeField | 自动 | 点赞时间 |

**约束**: `unique_together = ['user', 'reply']` — 同一用户对同一回复只能点赞一次。

## 路由配置

| URL | 视图 | HTTP | 需登录 | 缓存 | 说明 |
|-----|------|------|--------|------|------|
| `GET /forum/` | BoardListView | GET | 否 | **60 秒** | 版块列表 |
| `GET /forum/board/<slug>/` | TopicListView | GET | 否 | **30 秒** | 主题列表（指定版块，分页 20 条）|
| `GET /forum/board/<slug>/topic/<id>/` | TopicDetailView | GET | 否 | 无缓存 | 主题详情 + 回复列表 |
| `GET/POST /forum/board/<slug>/topic/create/` | topic_create_view | GET/POST | **是** | 无缓存 | 发新主题（限流检查由 moderation 处理）|
| `POST /forum/board/<slug>/topic/<id>/reply/` | reply_create_view | POST | **是** | 无缓存 | 发表回复（限流检查由 moderation 处理）|
| `POST /forum/reply/<id>/like/` | like_reply_view | POST | **是** | 无缓存 | 点赞/取消点赞回复 |

> ⚠️ 与评论系统不同，论坛主题和回复**没有独立的限流装饰器**，审核由 `smart_moderate_instance` 的 AI 审核负责。

## 视图层详解

### 📋 BoardListView — 版块列表

**缓存**: `@cache_page(60)` — 60 秒
**模板**: `forum/board_list.html`
**上下文**: `boards` — 所有版块列表（按名称排序）

**功能**: 展示所有版块及其统计信息（主题数、回复数、最后发帖时间）。

### 📑 TopicListView — 主题列表

**缓存**: `@cache_page(30)` — 30 秒
**模板**: `forum/topic_list.html`
**分页**: 20 条/页

**查询优化**:
```python
Topic.objects.filter(board=board, review_status='approved')
    .select_related('author', 'board')
    .order_by('-is_pinned', '-last_reply_at', '-created_at')
```

**上下文**: `topics`（主题列表）、`board`（当前版块信息）

### 📖 TopicDetailView — 主题详情

**无缓存**: 需要实时更新浏览量
**模板**: `forum/topic_detail.html`

**查询优化**:
```python
Topic.objects.filter(board=board, review_status='approved')
    .select_related('author', 'board')
    .prefetch_related(Prefetch(
        'replies',
        queryset=Reply.objects.filter(is_deleted=False, review_status='approved')
            .select_related('author')
            .only('id', 'content', 'created_at', 'like_count',
                  'author__id', 'author__username', 'author__nickname')
    ))
```

**行为**:
1. 获取主题对象
2. 调用 `increase_views()` 增加浏览数
3. 预加载已审核且未删除的回复
4. 填充 ReplyForm 供用户回复

**上下文**: `topic`（主题对象）、`replies`（回复列表）、`reply_form`（回复表单）

### ✍️ topic_create_view — 创建主题

**需登录**: `@login_required`
**模板**: `forum/topic_form.html`

```
POST /forum/board/<slug>/topic/create/ (需登录)
    ↓
获取 Board 对象（不存在 → 404）
    ↓
验证 TopicForm（title + content）
    ↓
创建 Topic（commit=False）
    ↓ topic.board = board
    ↓ topic.author = request.user
    ↓ topic.review_status = 'pending'
    ↓ topic.save()
    ↓
调用 smart_moderate_instance(topic)  → AI 智能审核
    ↓
    ├── approved → "主题已发布"
    ├── rejected → "主题未通审核：{message}"
    └── pending → "主题已提交，等待审核"
    ↓
board.update_counts() → 更新版块统计
    ↓
重定向到版块列表
```

### 💬 reply_create_view — 发表回复

**需登录**: `@login_required`
**只允许 POST**

```
POST /forum/board/<slug>/topic/<id>/reply/ (需登录)
    ↓
获取 Topic 对象（不存在或版块不匹配 → 404）
    ↓
检查是否锁定（topic.is_locked）
    ↓ 锁定 → HttpResponseForbidden('主题已被锁定，无法回复')
    ↓
验证 ReplyForm（content）
    ↓
创建 Reply（commit=False）
    ↓ reply.topic = topic
    ↓ reply.author = request.user
    ↓ reply.review_status = 'pending'
    ↓ reply.save()
    ↓
调用 smart_moderate_instance(reply)  → AI 智能审核
    ↓
topic.update_reply_count() → 更新主题回复数 + 最后回复时间 + 版块统计
    ↓
重定向到主题详情
```

### 👍 like_reply_view — 点赞回复

**需登录**: `@login_required`
**只允许 POST**

```
POST /forum/reply/<id>/like/ (需登录)
    ↓
获取 Reply 对象
    ↓
检查是否已审核通过（review_status == 'approved'）
    ↓ 未通过 → JSON {success: false, message: '只能对已审核通过的回复点赞'}
    ↓
ReplyLike.objects.get_or_create(user=request.user, reply=reply)
    ↓
    ├── created=True → 点赞 → liked = True
    └── created=False → 取消点赞 → liked = False
    ↓
reply.update_like_count()
    ↓
JSON: {success: true, liked: true/false, like_count: N, message: '操作成功'}
```

## 审核流程

```
用户提交主题/回复
    ↓
review_status = 'pending' (待审核)
    ↓
smart_moderate_instance(instance) → AI 智能审核
    ↓
    ├── approved → 自动通过，立即可见
    │   → Board.update_counts() (主题时)
    │   → Topic.update_reply_count() (回复时)
    │
    ├── rejected → 自动拒绝，附原因
    │   → 通知用户
    │
    └── pending → 进入人工审核队列
        → Moderator 仪表板显示
```

## 统计更新机制

### 版块统计 (`Board.update_counts()`)

```
获取所有已审核的主题
    ↓
topic_count = len(approved_topics)
    ↓
reply_count = SUM(topic.reply_count for topic in approved_topics)  ← 聚合查询
    ↓
last_post_at = 最新回复时间
    ↓
board.save()
```

### 主题统计 (`Topic.update_reply_count()`)

```
获取已审核且未删除的回复
    ↓
reply_count = len(approved_replies)
    ↓
last_reply_at = 最新回复时间
    ↓
topic.save()
    ↓
board.update_counts()  ← 级联更新版块统计
```

| # | 安全措施 | 实现方式 |

## 浏览量统计

与博客文章的 Redis 计数不同，论坛主题使用**简化的直接数据库更新**：

```python
def increase_views(self):
    Topic.objects.filter(pk=self.pk).update(views_count=F('views_count') + 1)
    # 内存值 +1（当前请求使用）
    if isinstance(self.views_count, int):
        self.views_count += 1
```

> **设计说明**: 没有使用 Redis 计数 + 异步同步，因为论坛模块通常流量小于博客文章。如果后期需要优化，可迁移到 Redis 计数方案。

## 数据库查询优化总结

| 查询 | 优化方式 |
|------|----------|
| BoardListView | 无特殊优化（Board 模型简单，只有 name/slug 等基础字段，数据量小）|
| TopicListView | `select_related('author', 'board')`，避免 N+1 |
| TopicDetailView | `select_related('author', 'board')` + `prefetch_related('replies', queryset=...)`，一次查询加载主题 + 所有审核通过的回复 + 作者信息 |
| Board.update_counts() | 聚合查询（`SUM`）替代 Python 循环统计 |
