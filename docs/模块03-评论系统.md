# 模块 03 — 评论系统

> 路径: `apps/blog/` 中的 Comment + CommentLike 模型 | URL前缀: `/blog/comment/` | 与博客模型共用文件

---

## 功能概览

文章评论的发表、审核、点赞系统。支持游客评论和登录用户评论的差异化处理，集成敏感词过滤 + AI 智能审核，防刷限流保护。

---

## 数据模型

### Comment（评论）

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| post | ForeignKey(Post) | **是** | - | 所属文章（`on_delete=CASCADE`，`related_name='comments'`）|
| user | ForeignKey(User) | 否 | `null` | 评论者（已登录用户自动关联，`on_delete=SET_NULL`，`related_name='comments'`）|
| name | CharField(50) | 否 | 空 | 评论者姓名（游客填登录时隐藏）|
| email | EmailField | 否 | 空 | 评论者邮箱（选填，登录时隐藏）|
| content | TextField | **是** | - | 评论内容 |
| review_status | CharField(20) | 是 | `pending` | 审核状态: pending(待审核)/approved(已通过)/rejected(已拒绝) |
| is_approved | BooleanField | 是 | `False` | **兼容字段**（与 `review_status` 自动同步，`review_status='approved'` → `is_approved=True`）|
| reviewed_by | ForeignKey(User) | 否 | `null` | 审核人（`on_delete=SET_NULL`，`related_name='reviewed_comments'`）|
| reviewed_at | DateTimeField | 否 | `null` | 审核时间 |
| review_note | TextField | 否 | 空 | 审核备注（拒绝时填写理由）|
| ip_address | GenericIPAddressField | 否 | `null` | 评论者 IP 地址 |
| user_agent | CharField(200) | 否 | 空 | 浏览器 UA 信息（截取前 200 字符）|
| like_count | PositiveIntegerField | 是 | `0` | 点赞数（自动更新）|
| created_at | DateTimeField | 自动 | 新增 | 创建时间 |
| updated_at | DateTimeField | 自动 | 新增 | 更新时间 |

### 索引

| 索引类型 | 字段 | 用途 |
|----------|------|------|
| 联合索引 | `post` + `created_at`(降序) | 文章下的评论列表查询（最常见的查询）|
| 常规索引 | `review_status` | 按审核状态筛选 |
| 常规索引 | `user` | 查询用户发表的所有评论 |
| 常规索引 | `created_at`(降序) | 全局按时间排序 |

### CommentLike（评论点赞）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user | ForeignKey(User) | **是** | 点赞用户（`on_delete=CASCADE`，`related_name='comment_likes'`）|
| comment | ForeignKey(Comment) | **是** | 被点赞的评论（`related_name='likes'`）|
| created_at | DateTimeField | 自动 | 点赞时间 |

**约束**: `unique_together = ['user', 'comment']` — 同一用户对同一评论只能点赞一次，数据库层面保证。

### 方法

| 方法 | 说明 |
|------|------|
| `__str__()` | 有 user 显示"`username 评论了 title`"，否则 "`name 评论了 title`" |
| `is_pending` (property) | 返回 `review_status == 'pending'` |
| `is_approved_status` (property) | 返回 `review_status == 'approved'` |
| `is_rejected` (property) | 返回 `review_status == 'rejected'` |
| `update_like_count()` | 从 `likes` 关系计算实际点赞数，回写到 `like_count`（`F()` 更新避免竞态）|
| `save()` | 同步 `is_approved` 与 `review_status`：`review_status='approved'` → `is_approved=True`，否则 `False` |

## 核心功能详解

### 📝 发表评论流程 (`comment_create_view`)

```
POST /blog/comment/<post_slug>/
    ↓
获取文章（Post.objects.get(slug=..., status='published')）
    ↓ 文章不存在或不是已发布 → 404
    ↓
验证 CommentForm
    ↓ 已登录用户 → 隐藏 name/email 字段
    ↓ 游客 → name 和 email 必填
    ↓ content 必填
    ↓
创建 Comment（commit=False）
    ↓ comment.post = post
    ↓ 登录用户 → comment.user = request.user
    ↓ 游客 → comment.name / comment.email 从表单获取
    ↓ comment.ip_address = request.META['REMOTE_ADDR']
    ↓ comment.user_agent = request.META['HTTP_USER_AGENT'][:200]
    ↓ comment.review_status = 'pending'（始终先设为待审核）
    ↓ comment.save()
    ↓
调用 smart_moderate_instance(comment)  → AI 智能审核
    ↓
    ├── status='approved' → "评论已发布"
    │   → 自动通过，立即可见
    │
    ├── status='rejected' → "评论未通审核：{message}"
    │   → AI 识别违规，自动拒绝
    │
    └── status='pending' → "评论已提交，等待审核后显示"
        → 进入人工审核队列
    ↓
重定向回文章详情页
```

### 👍 点赞评论流程 (`like_comment_view`)

```
POST /blog/comment/<comment_id>/like/ (需登录)
    ↓
获取 Comment 对象
    ↓
检查评论是否已审核通过（review_status == 'approved'）
    ↓ 未通过 → 返回 JSON {success: false, message: '只能对已审核通过的评论点赞'}
    ↓
CommentLike.objects.get_or_create(user=request.user, comment=comment)
    ↓
    ├── created=True → 新点赞（like 记录创建）→ liked = True
    └── created=False → 已赞，取消点赞（like 记录删除）→ liked = False
    ↓
comment.update_like_count() → 从 likes 关系计算实际点赞数，回写 like_count
    ↓
返回 JSON: {success: true, liked: true/false, like_count: N, message: '操作成功'}

```

## 表单详解

### CommentForm（评论表单）

| 字段 | 类型 | 登录用户 | 游客 | 说明 |
|------|------|----------|------|------|
| content | Textarea | **必填** | **必填** | 评论内容（4 行，`form-control`）|
| name | TextInput | **隐藏** | **必填** | 姓名（登录时自动删除字段）|
| email | EmailInput | **隐藏** | **必填** | 邮箱（登录时自动删除字段）|

**智能行为**:

```python
def __init__(self, *args, **kwargs):
    self.user = kwargs.pop('user', None)
    super().__init__(*args, **kwargs)

    if self.user and self.user.is_authenticated:
        # 已登录：隐藏姓名和邮箱（使用系统用户数据）
        del self.fields['name']
        del self.fields['email']
    else:
        # 游客：必填
        self.fields['name'].required = True
        self.fields['email'].required = True
```

**使用方式**:

```python
# 在视图中
form = CommentForm(user=request.user)

# 提交时
form = CommentForm(request.POST, user=request.user)
```

## 审核流程

### 完整审核路径

```
用户提交评论
    ↓
review_status = 'pending' (待审核)
    ↓
┌─────────────────────────────────────────────┐
│          smart_moderate_instance()           │
│                                              │
│  1. 敏感词检测（本地词库）                   │
│     ↓                                        │
│  2. 百度 AI 内容审核                         │
│     ↓                                        │
│  3. 判定结果：                              │
│     ├── approved → 自动通过（立即可见）     │
│     │   → reviewed_by = None                │
│     │   → reviewed_at = now()               │
│     │   → review_note = ''                  │
│     │   → ModerationLog(action='approved')  │
│     │                                        │
│     ├── rejected → 自动拒绝（附违规理由）   │
│     │   → review_status = 'rejected'        │
│     │   → review_note = '违规原因'         │
│     │   → ModerationLog(action='rejected')  │
│     │                                        │
│     └── pending → 进入人工审核队列          │
│         → review_status = 'pending'         │
│         → review_note = 'AI 评估结果'      │
│         → ModerationReminder 创建           │
└─────────────────────────────────────────────┘
    ↓
管理员审核（页面/API 两种模式）
    ↓
    ├── approve → review_status = 'approved'
    │   → reviewed_by = 管理员
    │   → ModerationLog(action='approved')
    │
    └── reject + review_note → review_status = 'rejected'
        → ModerationLog(action='rejected')
```

### 审核状态在前端的显示

| 状态 | 对用户可见？ | 说明 |
|------|--------------|------|
| `pending` | 仅作者和管理员可见 | 等待审核 |
| `approved` | **所有人可见** | 已通过审核，正常显示 |
| `rejected` | 仅作者可见（附拒绝理由）| 被拒绝 |

## 限流保护

| 场景 | 限流方式 | 阈值 | 依据 |
|------|----------|------|------| 10 条/分钟 | IP 地址 |
| 评论发表 | `@rate_limit('comment', rate='10/m')` | 10 条/分钟 | IP 地址 |
| 评论点赞 | `@rate_limit('like', rate='30/m')` | 30 次/分钟 | IP 地址 |

**目的**: 防止刷评、刷赞、垃圾评论泛滥。

## 个人中心集成

在 `profile_view`（用户个人中心）中，显示用户的评论状态：

```python
# 待审核评论
pending_comments = Comment.objects.filter(
    user=request.user, review_status='pending'
).order_by('-created_at')

# 被拒绝的评论
rejected_comments = Comment.objects.filter(
    user=request.user, review_status='rejected'
).order_by('-created_at')
```

用户可以查看：
- 自己发表的评论中哪些还在审核
- 哪些被拒（查看 `review_note` 了解原因）

## 数据库查询模式

### 文章详情页 — 预加载评论

```python
Post.objects.select_related('author', 'category')
    .prefetch_related(Prefetch(
        'comments',
        queryset=Comment.objects.filter(review_status='approved')
            .select_related('user')
            .only('id', 'content', 'name', 'email', 'like_count', 'created_at',
                  'user__id', 'user__username', 'user__nickname')
    ))
```

### 审核仪表板 — 待审核评论

```python
Comment.objects.filter(review_status='pending').order_by('-created_at')
```

### 审核 API — 已审核评论

```python
Comment.objects.filter(
    post=post, review_status='approved'
).select_related('user').order_by('created_at')
```

## 兼容性设计

| 旧字段 | 新字段 | 同步方式 |
|--------|--------|----------|
| `is_approved` (Boolean) | `review_status` (CharField) | `save()` 时同步：`review_status='approved'` → `is_approved=True`，否则 `False` |

> **为什么需要兼容**：可能旧版本使用了简单的 `is_approved` BooleanField，后来升级为更精细的 `review_status` 三态审核。保留 `is_approved` 可以避免代码库中所有使用旧字段的地方报错。

## 安全特性

| # | 安全措施 | 实现方式 |
|---|----------|----------|
| 1 | 防刷评论 | 限流 10 条/分钟（按 IP）|
| 2 | 防刷点赞 | 限流 30 次/分钟（按 IP）+ `unique_together` 约束 |
| 3 | IP 追溯 | 记录每条评论的 `ip_address` |
| 4 | 设备追溯 | 记录 `user_agent`（截取前 200 字符）|
| 5 | AI 审核 | 敏感词 + 百度 AI 双重检测 |
| 6 | 仅 POST 请求 | comment_create_view 和 like_comment_view 都不允许 GET |
| 7 | 只能对已审核评论点赞 | `if comment.review_status != 'approved': return error` |
| 8 | 兼容字段自动同步 | `save()` 自动同步 `is_approved` 与 `review_status` |

## 错误处理

| 场景 | 行为 |
|------|------|
| 文章不存在 | 404 页面 |
| 文章未发布 | 404 页面（`get_object_or_404(Post, slug=..., status='published')`）|
| 表单验证失败 | 重定向回文章页（不显示具体错误）|
| 未登录用户点赞 | Django 重定向到登录页（`@login_required`）|
| 对未审核评论点赞 | JSON 返回: `{success: false, message: '只能对已审核通过的评论点赞'}` |
| 重复点赞 | 自动转为取消点赞（toggle 行为）|
| AI 审核异常 | 标记为 pending，进入人工审核队列 |

## 性能优化

| 优化项 | 说明 |
|--------|------|
| `select_related('user')` | 预加载评论用户信息，避免 N+1 查询 |
| `.only(...)` | 只查询评论展示所需的字段（减少数据传输）|
| `prefetch_related` | 在文章详情中预加载所有已审核评论（一次查询搞定）|
| `unique_together` | 数据库层面保证不重复点赞（不需要额外查询判断）|
| F() 更新 | `update_like_count()` 使用 `F()` 更新避免竞态条件 |
