### 1.1 安全中间件 (`security_middleware.py`)

> ⚠️ **此中间件不包含 XSS 过滤或 SQL 注入防护**（这些由 Django ORM 和 SecurityMiddleware 负责），专注**路径遍历攻击检测**和**可疑请求日志**。

#### SecurityMonitorMiddleware — 安全监控中间件

```python
class SecurityMonitorMiddleware:
    SUSPICIOUS_PATTERNS = [
        '/etc/passwd', '/etc/shadow', '/windows/system32', '/proc/self',
        '../', '..\\',
        '.env', 'settings.py', 'config.py', 'secret',
        '.git', '.htaccess', 'web.config',
```

| 方法 | 功能 |
|------|------|
| `__call__(request)` | 正常传递请求（不拦截） |
| `process_exception(request, exception)` | 异常处理（当前返回 None）|

#### security_logger — 可疑请求日志中间件

```python
def security_logger(get_response):
    def middleware(request):
        path = request.path.lower()
        is_suspicious = any(pattern in path for pattern in [
            '/etc/', '/windows/', '/proc/', '../', '..\\',
            '.env', 'settings.py', 'config.py', 'web.config'
        ])
        if is_suspicious:
            logger.warning(
                f"[安全警告] 可疑请求: {request.method} {request.path} "
                f"IP: {request.META.get('REMOTE_ADDR', 'unknown')} "
                f"User-Agent: {request.META.get('HTTP_USER_AGENT', 'unknown')}"
            )
        return get_response(request)
    return middleware
```

记录：请求方法、路径、IP 地址、User-Agent。

#### custom_404_view — 自定义 404

```python
def custom_404_view(request, exception=None):
    path = request.path.lower()
    is_traversal_attempt = any(pattern in path for pattern in [
        '/etc/', '/windows/', '/proc/', '../', '..\\',
        '.env', 'settings.py'
    ])
    if is_traversal_attempt:
        logger.warning(f"[安全] 路径遍历尝试已阻止: {request.path}")
        return HttpResponseNotFound('<h1>404 Not Found</h1>')
    return render(request, '404.html', {'path': request.path}, status=404)
```

**双重处理**: 路径遍历攻击 → 简单纯文本 404（不暴露任何信息）| 正常 404 → 友好页面 `404.html`

#### custom_500_view — 自定义 500

### 1.3 CSP Nonce 中间件 (`csp_nonce.py`)

> 为每个请求生成**唯一的 32 位十六进制 nonce**，用于内联脚本和样式的安全执行。

#### CSPNonceMiddleware

```python
class CSPNonceMiddleware:
    NONCE_LENGTH = 16  # 16 字节 = 32 位十六进制

    def __init__(self, get_response):
        self.csp_enabled = getattr(settings, 'CSP_NONCE_ENABLED', False)
        self.report_only = getattr(settings, 'CSP_REPORT_ONLY', False)

    def __call__(self, request):
        request.csp_nonce = secrets.token_hex(self.NONCE_LENGTH)  # 生成唯一 nonce
        response = self.get_response(request)
        if self.csp_enabled:
            nonce = request.csp_nonce
            csp_policy = self._build_csp_with_nonce(nonce)
            if self.report_only:
                response['Content-Security-Policy-Report-Only'] = csp_policy
            else:
                response['Content-Security-Policy'] = csp_policy
        return response
```

#### CSP 策略（含 nonce）

```
default-src 'self'
script-src 'self' 'nonce-{32位}' https://cdn.jsdelivr.net
style-src 'self' 'nonce-{32位}' https://cdn.jsdelivr.net https://fonts.googleapis.com
img-src 'self' data: https: blob:
font-src 'self' data: https://fonts.gstatic.com
connect-src 'self'
frame-ancestors 'none'
form-action 'self'
base-uri 'self'
report-uri {CSP_REPORT_URI}  # 可选
```

#### 模板中使用

```html
<script nonce="{{ request.csp_nonce }}">
    // 安全内联脚本
</script>
<style nonce="{{ request.csp_nonce }}">
    /* 安全内联样式 */
</style>
```

#### 辅助函数

| 函数 | 说明 |
|------|------|
| `get_csp_nonce(request)` | 获取当前请求的 nonce |
| `csp_nonce_context(request)` | 模板 context processor（返回 `{'csp_nonce': ...}`）|

#### 配置项

|--------|--------|------|
| `CSP_NONCE_ENABLED` | False | 是否启用 CSP Nonce |

### 1.4 限流保护 (`rate_limit.py`)

> 这是模块 10 中**最复杂的文件**（8KB），使用 **Redis Lua 脚本**实现原子限流。

#### RATE_LIMIT_LUA_SCRIPT — Redis Lua 原子脚本

```lua
local key = KEYS[1]
local limit = tonumber(ARGV[1])
local window = tonumber(ARGV[2])
local current = tonumber(redis.call('GET', key) or "0")

if current >= limit then
    local ttl = redis.call('TTL', key)
    if ttl < 0 then ttl = window end
    return {current, ttl}  -- 返回当前计数 + 剩余时间
end

if current == 0 then
    redis.call('SET', key, 1, 'EX', window)
else
    redis.call('INCR', key)
end

local new_count = current + 1
local ttl = redis.call('TTL', key)
if ttl < 0 then ttl = window end
return {new_count, ttl}
```

**关键特性**:
- **原子操作**: check + increment 在 Lua 中原子执行，无竞态条件
- **SHA1 缓存**: 首次 `EVAL` → `SCRIPT LOAD` → 后续用 `EVALSHA`
- **自动降级**: Lua 失败 → `_fallback_rate_limit()`（Django cache get/set）

#### 函数清单

| 函数 | 说明 |
|------|------|
| `_execute_rate_limit_script(cache_key, limit, window)` | 执行 Lua 脚本 → 返回 `(count, ttl, is_limited)` |
| `_fallback_rate_limit(cache_key, limit, window)` | 降级模式：用 Django cache 的 get/set（非原子，容错用）|
| `_parse_rate(rate)` | 解析速率字符串 `10/m` → `(10, 60)`, `100/h` → `(100, 3600)` |
| `get_rate_limit_status(key_prefix, identifier, limit, window)` | 查询当前限流状态（仅查询，不计数）|

#### @rate_limit — 装饰器（IP + 用户双模式）

```python
@rate_limit('comment', rate='5/m', method='POST')
def comment_create_view(request):
    ...
```

**行为**:
- 已登录用户 → key: `ratelimit:comment:user:{user_id}`
- 未登录 → key: `ratelimit:comment:ip:{ip_address}`
- 仅限制指定 HTTP 方法（默认 `POST`）
- 超限时返回 **429**：

```json
{
    "error": "请求频率过快，请稍后重试",
    "retry_after": 45,
    "limit": 5,
    "window": "60s"
}
```

响应头：`Retry-After: 45`

#### @rate_limit_by_user — 仅限已登录用户

```python
@rate_limit_by_user('post', rate='5/h')
def post_create_view(request):
    ...
```

- 未登录用户 → 跳过限流
- 已登录 → key: `ratelimit:post:user:{user_id}`

#### 速率格式

| 格式 | 说明 |
|------|------|
| `10/s` | 10 次/秒 |
| `10/m` | 10 次/分钟 |
| `100/h` | 100 次/小时 |
| `1000/d` | 1000 次/天 |

## 七、错误处理

| 场景 | 行为 |
|------|------|
| CSP 未启用 | 不加响应头（零影响）|
| Lua 脚本 SHA1 失效 | 自动回退 EVAL 并重新 SCRIPT LOAD |
| Lua 脚本完全失败 | 降级为 Django cache get/set（非原子）|
| Redis 连接异常 | 错误日志 + 返回降级结果 |
| 数据库连接不可用 | ConnectionPoolMiddleware 自动关闭并重连 |
| Session 清理失败 | 返回 0，记录异常日志 |
| 缓存预热失败 | 单项标记 error，不影响其他项 |
