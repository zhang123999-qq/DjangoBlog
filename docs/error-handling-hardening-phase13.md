# 异常处理分层收敛（阶段 13：拒绝原因校验 + 违规类型）

覆盖文件：
- `moderation/templates/moderation/dashboard.html`

## 目标

继续强化审核拒绝输入质量，减少“无意义备注/缺分类”问题。

## 本次实现

1. 拒绝弹层新增 `违规类型` 下拉：
   - 广告/推广
   - 敏感内容
   - 辱骂/攻击
   - 低质灌水
   - 其他

2. 拒绝原因校验升级：
   - 必须选择违规类型
   - 拒绝原因必须 4-200 字

3. 提交格式统一：
   - 自动拼接为 `[违规类型] 拒绝原因`

4. 本地记忆保持：
   - 最近原因仍按“正文原因”保存与复用（不含前缀）

## 效果

- 审核备注结构更清晰
- 后续统计/检索可按类型前缀聚合
- 减少过短/无效拒绝原因
