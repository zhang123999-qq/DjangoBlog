# 异常处理分层收敛（阶段 23：失败退避重试与连续失败告警）

覆盖文件：
- `moderation/templates/moderation/dashboard.html`

## 目标

提高指标面板在异常网络/接口抖动场景下的韧性，避免固定频率盲打。

## 实现

1. 连续失败计数
   - `failStreak` 记录连续失败次数（HTTP 失败与异常都计入）

2. 指数退避重试
   - 基于当前刷新间隔作为 base
   - `delay = min(base * 2^(failStreak-1), 120000ms)`
   - 自动刷新关闭时不触发退避重试

3. 连续失败告警
   - 连续失败达到 3 次后 toast 告警
   - 同一失败次数只提示一次，避免刷屏

4. 成功恢复
   - 拉取成功后重置 `failStreak`、清理 backoff 计时器

## 说明

- 仅前端容错增强，不影响主审核操作链路。
