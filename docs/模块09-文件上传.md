# 模块 09 — 文件上传

> 核心文件: `apps/core/upload_views.py` + `apps/core/upload_tasks.py` | URL前缀: `/api/upload/` | 框架: DRF API-views

---

## 功能概览

安全的文件和图片上传系统，支持**同步**和**异步**两种模式，集成**多层安全校验**（MIME 类型、文件扩展名、文件头 Magic 校验、ClamAV 病毒扫描）。异步模式下使用**隔离区存储**方案，文件先存入隔离区，经 Celery 任务扫描通过后才移动到正式目录。

## 安全机制（图片上传）

### upload_image — 6层安全校验

文件进入 → 逐层校验 → 任何一层失败即拒绝

| 校验层 | 校验内容 | 通过条件 | 失败错误码 |
|--------|----------|----------|-----------|
| 1. 存在性 | `request.FILES.get("file")` | 文件存在 | `UPLOAD_NO_FILE` |
| 2. 文件大小 | `upload.size <= 5*1024*1024` | ≤ 5MB | `UPLOAD_IMAGE_TOO_LARGE` |
| 3. MIME 类型 | `upload.content_type` 在白名单中 | `image/jpeg/png/gif/webp` 四选一 | `UPLOAD_IMAGE_TYPE_NOT_ALLOWED` |
| 4. 扩展名 | `ext` 在白名单中 | `jpg/jpeg/png/gif/webp` | `UPLOAD_IMAGE_TYPE_NOT_ALLOWED` |
| 5. Magic 校验 | 文件头 32 字节匹配图片签名 | 见下表 | `UPLOAD_IMAGE_MAGIC_INVALID` |
| 6. 危险 Magic 检测 | 文件头不能是 PE/ELF/Shebang | 不以 `MZ`/`\x7fELF`/`#!` 开头 | `UPLOAD_IMAGE_MAGIC_INVALID`（隐含）|
| 7. ClamAV 扫描 | 病毒扫描（可配置） | `OK` 或 fail-open | `UPLOAD_SECURITY_SCAN_REJECTED` |

### Magic 签名对照表

| 扩展名 | Magic 字节签名 | 说明 |
|--------|----------------|------|
| **JPG/JPEG** | `\xff\xd8\xff` | SOI (Start Of Image) 标记 |
| **PNG** | `\x89PNG\r\n\x1a\n` | PNG 文件头（8字节）|
| **GIF** | `GIF87a` 或 `GIF89a` | GIF87a / GIF89a 版本 |
| **WEBP** | `RIFF`...`WEBP` | RIFF 容器，第 9-12 字节为 `WEBP` |

> **WebP 特殊处理**: `_looks_like_webp()` 检查前 12 字节，要求 ≥12 字节、开头 `RIFF`、第 9-12 字节 `WEBP`。

### 危险 Magic 检测

| 危险头字节 | 文件类型 | 检测方法 |
|------------|----------|----------|
| `MZ` (`\x4d\x5a`) | Windows PE 可执行文件 | `header.startswith(b"MZ")` |
| `\x7fELF` | Linux ELF 可执行文件 | `header.startswith(b"\x7fELF")` |
| `#!` | Unix Shebang（脚本） | `header.startswith(b"#!")` |

## 安全机制（文件上传）

### upload_file — 5层安全校验 + 双模式（同步/异步）

| 校验层 | 校验内容 | 通过条件 | 失败错误码 |
|--------|----------|----------|-----------|
| 1. 存在性 | `request.FILES.get("file")` | 文件存在 | `UPLOAD_NO_FILE` |
| 2. 文件大小 | `upload.size <= 10*1024*1024` | ≤ 10MB | `UPLOAD_FILE_TOO_LARGE` |
| 3. 危险扩展名 | `ext` 不在黑名单中 | 不在 `DANGEROUS_FILE_EXTENSIONS` | `UPLOAD_FILE_TYPE_DENIED` |
| 4. 允许扩展名 | `ext` 在白名单中 | 在 `FILE_ALLOWED_EXTENSIONS` | `UPLOAD_FILE_EXT_NOT_ALLOWED` |
| 5. 危险 Magic 检测 | 文件头不能是 PE/ELF/Shebang | 不以 `MZ`/`\x7fELF`/`#!` 开头 | `UPLOAD_FILE_MAGIC_DENIED` |
| 6. ClamAV 扫描 | 病毒扫描 | `OK` 或 fail-open | `UPLOAD_SECURITY_SCAN_REJECTED` |

### 危险扩展名黑名单（18种）

| 类型 | 扩展名 |
|------|--------|
| **Windows 可执行** | `exe`, `dll`, `msi`, `scr`, `com`, `bat`, `cmd` |
| **Linux 脚本** | `sh` |
| **脚本语言** | `php`, `phtml`, `jsp`, `asp`, `aspx`, `cgi`, `pl`, `py`, `rb` |
| **Java** | `jar` |

### 支持的文件类型（白名单 20 种）

| 类型 | 扩展名 |
|------|--------|
| **图片** | `jpg`, `jpeg`, `png`, `gif`, `webp` |
| **文档** | `pdf`, `txt`, `md`, `csv`, `json`, `doc`, `docx`, `xls`, `xlsx`, `ppt`, `pptx` |
| **压缩** | `zip`, `7z`, `rar` |

## 视图层详解

### 🔧 upload_image — 图片上传（同步模式）

```
POST /api/upload/image/
    ↓
1. 校验文件存在 → 无 → 400 (UPLOAD_NO_FILE)
    ↓
2. 校验文件大小 ≤ 5MB → 超 → 400 (UPLOAD_IMAGE_TOO_LARGE)
    ↓
3. 提取扩展名 → _safe_extension(upload.name)
    ↓
4. 校验 MIME 类型 + 扩展名 → 不匹配 → 400 (UPLOAD_IMAGE_TYPE_NOT_ALLOWED)
    ↓
5. Magic 校验 → _validate_image_magic(upload, ext) → 失败 → 400 (UPLOAD_IMAGE_MAGIC_INVALID)
    ↓
6. ClamAV 扫描 → _scan_with_clamav(upload) → 失败 → 400 (UPLOAD_SECURITY_SCAN_REJECTED)
    ↓
7. 保存文件 → _save_upload(upload, "images") → 成功 → 200 {location: url}
    ↓ 异常 → 500 (UPLOAD_SAVE_FAILED)
```

**装饰器**:
```python
@extend_schema(operation_id="upload_image", summary="上传图片（同步模式）", tags=["upload"], ...)
@api_view(["POST"])
@permission_classes([permissions.IsAuthenticated])
@throttle_classes([UploadRateThrottle])
@parser_classes([MultiPartParser, FormParser])
```

### 📁 upload_file — 文件上传（同步/异步双模式）

```
POST /api/upload/file/
    ↓
1-3. 同 upload_image（文件存在 → 大小 ≤ 10MB → 提取扩展名）
    ↓
4. 校验危险扩展名 → 在黑名单 → 400 (UPLOAD_FILE_TYPE_DENIED)
    ↓
5. 校验允许扩展名 → 不在白名单 → 400 (UPLOAD_FILE_EXT_NOT_ALLOWED)
    ↓
6. 危险 Magic 检测 → _contains_dangerous_magic(upload, ext) → 失败 → 400 (UPLOAD_FILE_MAGIC_DENIED)
    ↓
    ╔═══════════════════════════════════════════╗
    ║  UPLOAD_ASYNC_PIPELINE_ENABLED = False    ║
    ║  同步模式: ClamAV 扫描 → 保存 → 返回 200  ║
    ║  {location, filename, size}               ║
    ╠═══════════════════════════════════════════╣
    ║  UPLOAD_ASYNC_PIPELINE_ENABLED = True     ║
    ║  异步模式: 保存隔离区 → 触发 Celery       ║
    ║  → 返回 202 {status, upload_id, task_id,  ║
    ║  status_path}                             ║
    ╚═══════════════════════════════════════════╝
```

**装饰器**:
```python
@extend_schema(operation_id="upload_file", summary="上传通用文件（同步/异步）", tags=["upload"], ...)
@api_view(["POST"])
@permission_classes([permissions.IsAuthenticated])
@throttle_classes([UploadRateThrottle])
@parser_classes([MultiPartParser, FormParser])
```

### 📊 upload_status — 查询异步上传状态

```
GET /api/upload/status/<upload_id>/
    ↓
cache.get(f"upload:status:{upload_id}")
    ↓ None → 404 (UPLOAD_TASK_NOT_FOUND)
    ↓ 存在 → 200 {status, upload_id, location?, reason?}
```

**返回值**:
```json

    "status": "pending" | "scanning" | "ready" | "rejected" | "failed",
    "upload_id": "abc123...",
    "location": "/media/uploads/files/2026/04/xxx.pdf",  // ready 状态才有
    "reason": "clamav: virus found",                     // rejected 状态才有
    "filename": "original_name.pdf"
}
```

**装饰器**:
```python
@extend_schema(operation_id="upload_status", summary="查询异步上传状态", tags=["upload"], ...)
@api_view(["GET"])
@permission_classes([permissions.IsAuthenticated])
```

## 工具函数

### _safe_extension — 安全提取扩展名

```python
def _safe_extension(filename: str) -> str:
    if "." not in filename:
        return ""
    return filename.rsplit(".", 1)[-1].lower().strip()
```

> 处理无扩展名文件名，防止 `.rsplit()` 异常。

### _peek_header — 安全读取文件头

```python
def _peek_header(upload, size: int = 32) -> bytes:
    pos = upload.tell()  # 记录当前位置
    header = upload.read(size)
    upload.seek(pos)  # 恢复位置（避免影响后续读取）
    return bytes(header)
```

### _validate_image_magic — 图片 Magic 校验

| 扩展名 | Magic 签名 | 长度要求 |
|--------|-----------|----------|
| jpg/jpeg | `\xff\xd8\xff` | 3 字节 |
| png | `\x89PNG\r\n\x1a\n` | 8 字节 |
| gif | `GIF87a` 或 `GIF89a` | 6 字节 |
| webp | `RIFF`...`WEBP` | 12 字节 |

### _contains_dangerous_magic — 危险头检测

| 头字节 | 文件类型 | 风险 |
|--------|----------|------|
| `MZ` | Windows PE 可执行 | 极高 |
| `\x7fELF` | Linux ELF 可执行 | 极高 |
| `#!` | Unix Shebang 脚本 | 中等 |

### _save_upload — 保存上传文件

```python
def _save_upload(upload, folder: str) -> str:
    ext = _safe_extension(upload.name)
    today = datetime.now().strftime("%Y/%m")
    filename = f"{uuid.uuid4().hex}.{ext}" if ext else uuid.uuid4().hex
    relative_path = f"uploads/{folder}/{today}/{filename}"
    saved_path = default_storage.save(relative_path, upload)
    return f"{settings.MEDIA_URL}{saved_path}"
```

### ClamAV 扫描函数

| 函数 | 参数类型 | 用途 |
|------|----------|------|
| `_clamav_scan_fileobj(file_obj)` | 文件对象 | upload_views.py 使用（同步扫描）|
| `_clamav_scan_bytes(content)` | bytes | upload_tasks.py 使用（异步扫描）|

**ClamAV 通信协议**: `zINSTREAM` 协议
```
连接 → 发送 `zINSTREAM\0` 标志
    ↓ 发送 chunk: [4-byte length][chunk data] → 直到 length=0
    ↓ 接收响应: "OK" / "FOUND: virusname" / 其他
```

**容错策略**:
- `CLAMAV_FAIL_CLOSED = True` → ClamAV 异常时拒绝上传（安全优先）
- `CLAMAV_FAIL_CLOSED = False` → ClamAV 异常时通过上传（体验优先，默认）

## Celery 异步任务 (upload_tasks.py) 🆕

> 这是**异步上传模式的核心**，文件先存入隔离区，经扫描通过后才移动到正式目录。

### process_quarantined_upload — 隔离文件扫描及转换

```python
@shared_task(bind=True, autoretry_for=(Exception,), retry_backoff=True, retry_kwargs={'max_retries': 2})
def process_quarantined_upload(self, upload_id, quarantine_path, ext, original_name):
```

**重试配置**:
- `autoretry_for=(Exception,)` — 所有异常自动重试
- `retry_backoff=True` — 指数退避（1s → 2s → 4s）
- `max_retries=2` — 最多重试 2 次

**状态机（5种状态）**:

```
pending (创建任务时)
    ↓
scanning (开始执行)
    ↓
    ├── failed (隔离文件丢失) → 结束
    ├── rejected (ClamAV 不通过) → 删除隔离文件 → 结束
    └── ready (通过) → 保存到正式目录 → 删除隔离文件 → 结束
```

**详细流程**:
```
1. 标记状态: scanning
    ↓
2. 检查隔离文件是否存在 → 不存在 → failed → return
    ↓
3. 读取文件全部内容为 bytes
    ↓
4. ClamAV 字节流扫描 → _clamav_scan_bytes(content)
    ↓
    ├── 不通过:
    │   ├── 删除隔离文件 default_storage.delete(quarantine_path)
    │   ├── 标记状态: rejected, reason=detail
    │   └── return {'ok': False, 'reason': detail}
    │
    └── 通过:
        ├── 生成新文件名: {uuid4().hex}.{ext}
        ├── 保存到正式目录: uploads/files/YYYY/MM/{filename}
        ├── 删除隔离文件 default_storage.delete(quarantine_path)
        ├── 标记状态: ready, location=url
        └── return {'ok': True, 'location': location}
```

### 异步上传完整流程图

```
用户 POST /api/upload/file/
    ↓
安全检查（大小、扩展名、Magic）
    ↓
保存到隔离区: uploads/quarantine/YYYY/MM/{upload_id}.{ext}
    ↓
触发 Celery: process_quarantined_upload.delay(...)
    ↓
返回 202:
{
    "status": "pending",
    "upload_id": "abc123",
    "task_id": "celery-task-uuid",
    "status_path": "/api/upload/status/abc123/"
}

用户轮询 GET /api/upload/status/abc123/
    ↓ 状态: pending
    ↓ Celery 任务开始 → scanning
    ↓ ClamAV 扫描 →
    │   ├── rejected: 文件有病毒 → 删除 → 状态: rejected
    │   └── ready: 文件安全 → 移动 → 状态: ready, location=url
```

## 存储路径

```
media/uploads/images/YYYY/MM/<uuid>.<ext>       # 图片（同步模式）
media/uploads/files/YYYY/MM/<uuid>.<ext>        # 文件（同步模式 + 异步模式就绪后）
media/uploads/quarantine/YYYY/MM/<uuid>.<ext>   # 隔离区（异步模式暂存）
```

**路径生成规则**:
- **UUID 作为文件名**: 防止文件名冲突和路径遍历攻击
- **按年月分类**: `YYYY/MM/` 格式，便于管理和清理
- **保留原始扩展名**: 通过 `_safe_extension()` 安全提取
