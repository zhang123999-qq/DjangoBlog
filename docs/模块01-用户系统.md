# 模块 01 — 用户系统

> 路径: `apps/accounts/` | URL前缀: `/accounts/` | 独立应用模块

---

## 功能概览

全站用户身份认证中心。负责用户注册、登录、登出、个人资料管理，提供验证码防刷机制、自定义用户模型、随机头像自动分配、个人审核状态查看等完整功能。

---

## 数据模型

### User（自定义用户模型）

继承自 `AbstractUser`，扩展了以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | CharField(150) | 是 | 继承 `AbstractUser`，用户名 |
| email | EmailField(254) | 是 | **唯一约束**（覆盖默认非唯一行为）|
| nickname | CharField(50) | 否 | 用户昵称 |
| first_name | CharField(150) | 否 | 继承 `AbstractUser` |
| last_name | CharField(150) | 否 | 继承 `AbstractUser` |
| password | CharField(128) | 是 | 继承 `AbstractUser`（PBKDF2 + SHA256 哈希）|
| is_active | BooleanField | 是 | 继承 `AbstractUser`，是否可用 |
| is_staff | BooleanField | 是 | 继承 `AbstractUser`，能否进入后台 |
| is_superuser | BooleanField | 是 | 继承 `AbstractUser`，超级管理员 |
| date_joined | DateTimeField | 是 | 继承 `AbstractUser`，注册时间 |
| last_login | DateTimeField | 否 | 继承 `AbstractUser`，最后登录时间 |

> **关键配置**: 在 settings 中通过 `AUTH_USER_MODEL = 'accounts.User'` 指定全局使用此自定义模型。

### Profile（用户资料）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user | OneToOneField(User) | 是 | 与 User 一对一关联，`related_name='profile'`，删除 User 时级联删除 Profile |
| avatar | ImageField | 否 | 头像文件（上传到 `avatars/` 目录）|
| bio | TextField | 否 | 个人简介 |
| website | URLField | 否 | 个人网站链接 |
| created_at | DateTimeField | 自动 | 创建时间 |
| updated_at | DateTimeField | 自动 | 更新时间 |

## 核心功能详解

###  用户注册流程

```
页面加载 (GET)
    ↓ 生成 6 位数字验证码（secrets 模块）
    ↓ 创建验证码图片（PIL：干扰线 + 噪点 + 文字）
    ↓ Base64 编码后返回给前端
    ↓ 验证码保存到 session
    ↓
用户提交 (POST)
    ↓ 验证 UserRegisterForm（用户名唯一、邮箱唯一、密码强度、验证码）
    ↓ 验证通过 → form.save() 创建 User
    ↓ 自动触发 post_save 信号 → 创建 Profile + 分配随机头像
    ↓ 自动登录（指定认证后端）→ redirect 到首页
    ↓ 验证失败 → 重新生成验证码 → 返回表单 + 错误提示
```

**注册表单字段**: `username`, `email`, `nickname`, `password1`, `password2`, `captcha`

**表单校验**:
- `UserCreationForm` 内置：密码一致性、密码强度（长度≥8、不能是常见密码、不能纯数字）
- `clean_captcha()`：验证码比对 → 校验成功后立即销毁（防止重放攻击）

###  用户登录流程

```
页面加载 (GET)
    ↓ 生成验证码 → 保存 session → 显示登录表单
    ↓
用户提交 (POST)
    ↓ 验证 CustomLoginForm（用户名/密码/验证码）
    ↓ 验证码校验 → validate_captcha()
    ↓ 调用 authenticate(request, username, password)
    ↓ 认证成功 → login() → redirect 首页
    ↓ 认证失败 → 重新生成验证码 → 返回表单 + 错误提示
```

**登录表单**: 继承 `AuthenticationForm`，添加了 `captcha` 字段。
- `clean_captcha()` 在密码校验前执行，验证码错误直接拦截请求
- 所有输入框自动添加 Bootstrap `form-control` 样式

### 🚪 用户登出

```
POST /accounts/logout/ (仅允许 POST)
    ↓ logout(request) 清除认证信息
    ↓ redirect 首页
```

**安全考量**: 仅允许 POST 请求，避免被恶意链接触发登出。如果收到 GET 请求，返回 `HttpResponseNotAllowed`。

### 👤 个人中心

```
@ login_required 保护
    ↓
加载用户数据：
├── pending_comments: 待审核的评论（按时间倒序）
├── pending_topics: 待审核的主题
├── pending_replies: 待审核的回复
├── rejected_comments: 被拒绝的评论（含拒绝理由）
├── rejected_topics: 被拒绝的主题
└── rejected_replies: 被拒绝的回复
```

**用户可通过个人中心查看**:
- 自己发表但尚未通过审核的内容
- 被拒绝的内容（查看审核备注，了解拒绝原因）
- 个人资料和头像

### ✏️ 编辑个人资料

```
@ login_required 保护
    ↓
GET: 填充当前数据
    ↓ UserUpdateForm: username, email, nickname（排除 password 字段）
    ↓ ProfileUpdateForm: avatar, bio, website
    ↓
POST: 验证 → 保存
    ↓ user_form.save() + profile_form.save()
    ↓ redirect 回到个人中心
    ↓
头像验证：大小限制 1MB，超过则拒绝上传
```

## 验证码系统

### 技术实现 (`captcha.py`)

| 特性 | 说明 |
|------|------|
| 验证码长度 | 6 位数字 |
| 随机源 | `secrets` 模块（密码学安全，非 `random`）|
| 组合空间 | 1,000,000 种可能 |
| 图片格式 | PNG，120×40 像素 |
| 干扰手段 | 5 条干扰线 + 50 个噪点 |
| 字体 | `arial.ttf` 30px（降级 `ImageFont.load_default()`）|
| 返回方式 | Base64 编码字符串（直接内嵌 `src="data:image/png;base64,..."`）|

### 安全特性

1. **使用 `secrets` 替代 `random`**: `random` 是可预测的伪随机数，攻击者可以暴力破解。`secrets` 使用系统级熵源，不可预测。
2. **一次性使用**: 验证成功后立即从 session 中删除，防止重放攻击。
3. **验证码不可见**: 图片不保存为文件，每次动态生成，避免被爬虫批量下载。

## 头像系统 (`avatar_utils.py`)

### 随机头像分配

```
新用户注册 (post_save 信号触发)
    ↓ get_random_avatar()
    ↓ 扫描 static/img/avatars/ 目录
    ↓ 筛选 .png/.jpg/.jpeg/.gif 文件
    ↓ 随机选择一张
    ↓ 复制到 media/avatars/ 目录
    ↓ 生成 UUID 文件名（避免冲突）：{uuid_hex[:8]}.{ext}
    ↓ 返回 ImageField 路径：avatars/{uuid}.png
```

### 安全降级

| 情况 | 降级策略 |
|------|----------|
| 没有头像文件 | 返回 `avatars/default-avatar.png` |
| shutil.copy2 失败 | 返回 `avatars/default-avatar.png` |

### 辅助函数

| 函数 | 返回值 | 用途 |
|------|--------|------|
| `get_avatar_list()` | `[str, ...]` | 列出 `media/avatars/` 中所有头像 |
| `get_avatar_count()` | `int` | 统计可用头像数量 |

## 信号机制 (Signals)

```python
@receiver(post_save, sender=User, dispatch_uid='create_user_profile')
def create_or_update_user_profile(sender, instance, created, **kwargs): ...
```

| 触发时机 | 条件 | 行为 |
|----------|------|------|
| 用户创建 | `created=True` | 创建 Profile → 调用 `get_random_avatar()` → 分配随机头像 → 保存 Profile |
| 用户更新 | `created=False` | 确保 Profile 存在（`get_or_create`，防止意外删除 Profile）|

**`dispatch_uid`**: 避免信号被重复注册（Django reload 时信号可能被多次挂载）。

## 表单汇总

| 表单类 | 基类 | 字段 | 用途 |
|--------|------|------|------|
| `CustomLoginForm` | `AuthenticationForm` | username, password, captcha | 用户登录 |
| `UserRegisterForm` | `UserCreationForm` | username, email, nickname, password1, password2, captcha | 用户注册 |
| `UserUpdateForm` | `UserChangeForm` | username, email, nickname（排除 password）| 编辑用户信息 |
| `ProfileUpdateForm` | `ModelForm` | avatar, bio, website | 编辑个人资料 |

### 表单校验规则

| 规则 | 说明 |
|------|------|
| 密码一致性 | `password1` 必须等于 `password2` |
| 密码强度 | 长度 ≥ 8，不能与用户名相同，不能是常见密码 |
| 邮箱唯一 | `email` 字段唯一约束 |
| 用户名唯一 | `username` 字段唯一约束（Django 内置）|
| 验证码 | 与 session 中的 `captcha_code` 比较，匹配后销毁 |
| 头像大小 | 不超过 1MB |

## 安全清单

| # | 安全措施 | 实现方式 |
|---|----------|----------|
| 1 | 防暴力破解（验证码） | `secrets` 模块 6 位数字，一次性使用 |
| 2 | 防重放攻击 | 验证码验证后立即销毁 |
| 3 | 密码哈希 | PBKDF2 + SHA256（Django 默认）|
| 4 | 防 CSRF | Django CSRF 中间件 |
| 5 | 登出安全 | 仅允许 POST 请求 |
| 6 | 头像大小限制 | 表单校验 1MB |
| 7 | 信号防重复 | `dispatch_uid` 防止重复注册 |
