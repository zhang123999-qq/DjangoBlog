## 架构设计

### 工具注册机制

```
启动/首次访问
    ↓
ToolRegistry.get_tool(slug) / get_all_tools()
    ↓
registry._discovered == False → discover_tools()
    ↓
遍历 tool_modules/*.py
    ↓ importlib.import_module()
    ↓
查找 BaseTool 的子类
    ↓
tool_instance = obj()（实例化）
    ↓
registry.tools[slug] = tool_instance
    ↓
registry._discovered = True（只执行一次）
    ↓
Django 缓存 (1 小时) / registry.tools 内存缓存
```

### 工具生命周期

1. **自动发现**: 遍历 `tool_modules/` 目录，自动导入所有 `.py` 文件
2. **类注册**: 查找继承自 `BaseTool` 的子类并实例化
3. **Slug 去重**: 确保每个工具 slug 唯一（重复时抛出 `ValueError`）
4. **懒加载**: 访问工具时才真正加载到内存
5. **缓存**: 全量工具列表缓存 1 小时，分类缓存 1 小时
6. **配置管理**: 通过 `ToolConfig` 数据库表控制启用/禁用和排序

### 错误处理

| 场景 | 行为 |
|------|------|
| 工具模块导入失败 | `logger.warning()` 记录，继续加载其他工具 |
| 工具目录不存在 | 自动创建目录，跳过发现 |
| slug 重复 | 抛出 `ValueError`（严重错误）|
| 缓存失效 | 重新发现工具 |

### 重置机制

| 方法 | 说明 |
|------|------|
| `registry.clear_cache()` | 清除工具缓存（Django 缓存）|
| `registry.reset_discovered()` | 重置发现状态 + 清除所有缓存 + 清空工具列表。下次访问重新发现。|

## 工具基类 (`base_tool.py`)

所有工具必须继承 `BaseTool`，实现以下属性和方法：

### 类属性（ClassVar）

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| `name` | str | `"工具"` | 否 | 工具显示名称 |
| `slug` | str | `"tool"` | **是（应重写）** | 工具唯一标识（URL 路径）|
| `description` | str | `"工具描述"` | 否 | 工具描述 |
| `icon` | str | `"bi-tools"` | 否 | Bootstrap Icons 图标类名 |
| `category` | str | `ToolCategory.OTHER` | 否 | 分类键（见分类表）|
| `template_name` | str | `"tools/tool_detail.html"` | 否 | 工具模板路径 |
| `form_class` | Form class | `None` | 否 | Django Form 类（无表单则为 None）|

### 方法

| 方法 | 说明 | 子类是否需重写 |
|------|------|----------------|
| `get_form(data=None, files=None)` | 返回表单实例。如果有 `form_class` 则返回表单对象，否则返回 `None` | 否（默认即可）|
| `get_context(request, form=None, result=None)` | 返回字典：`{'tool', 'form', 'result', 'category_info'}` | 否（可扩展）|
| `handle(request, form)` | 处理工具逻辑，返回结果。抛出 `NotImplementedError` 如果未实现 | **是**（必须重写）|

### 工作流程

```
用户访问 /tools/<slug>/ → tool_detail 视图
    ↓
registry.get_tool(slug) → 获取工具实例
    ↓
检查 ToolConfig.is_enabled
    ↓ 禁用 → 返回错误页面
    ↓
GET 请求:
    ↓ form = tool.get_form() → 渲染空表单 + 模板
    ↓
POST 请求:
    ↓ form = tool.get_form(request.POST)
    ↓ 验证 form.is_valid()
    ↓ result = tool.handle(request, form)
    ↓ context = tool.get_context(request, form, result)
    ↓ render(request, tool.template_name, context)
```

## 视图层详解

### 🔧 tool_list — 工具列表

**缓存**: `@cache_page(300)` — 5 分钟
**模板**: `tools/tool_list.html`

**上下文**:
```python
{
    'categories': registry.get_categories_with_tools(),  # 按分类组织
    'tools': registry.get_all_tools(),                   # 所有工具列表
    'total_tools': len(all_tools),                       # 工具总数
    'enabled_tools_count': len(all_tools),               # 已启用数量
    'category_info': TOOL_CATEGORIES,                    # 分类配置

```

### 🔧 tool_detail — 工具详情/运行

**无缓存**: 需要实时交互

**流程**:
```
registry.get_tool(tool_slug)
    ↓ 工具不存在 → 渲染 tool_list.html，显示错误消息
    ↓
检查 ToolConfig.is_enabled（缓存 5 分钟）
    ↓ 禁用 → 渲染 tool_list.html，显示错误消息
    ↓
GET 请求: form = tool.get_form()
    ↓
POST 请求:
    ↓ form = tool.get_form(request.POST)
    ↓ form.is_valid() → True
    ↓ result = tool.handle(request, form)
    ↓
context = tool.get_context(request, form, result)
    ↓
render(request, tool.template_name, context)
```

### 🌍 my_ip_json — 返回访问者 IP

**用途**: NAT 检测工具的前端需要获取公网 IP。

```python

ip = tool._get_client_ip(request)
return JsonResponse({'ok': True, 'ip': ip})
```

### 懒加载 vs 全量缓存

| 机制 | 触发时机 | 行为 |
|------|----------|------|
| `get_tool(slug)` | 访问 `/tools/<slug>/` | 只加载请求的工具，不触发全量发现 |
| `get_all_tools()` | 访问 `/tools/` | 缓存命中 → 直接返回；未命中 → discover_tools() → 缓存 1 小时 |
| `get_categories_with_tools()` | 访问 `/tools/` | 从缓存获取或重新发现 |

## 开发新工具指南

### 最少步骤（3 步）

```python
# apps/tools/tool_modules/my_new_tool.py
from apps.tools.base_tool import BaseTool
from django import forms

class MyNewForm(forms.Form):
    text = forms.CharField(label='输入文本', widget=forms.Textarea(attrs={'rows': 4}))

class MyNewTool(BaseTool):
    name = "我的新工具"
    slug = "my-new-tool"
    description = "这是一个示例工具"
    icon = "bi-stars"  # Bootstrap Icons
    category = "text"   # 使用 categories.py 中定义的分类键
    form_class = MyNewForm

    def handle(self, request, form):
        text = form.cleaned_data['text']
        return {
            'result': text.upper(),
            'message': f"处理了 {len(text)} 个字符"
        }
```

### 注意事项
1. **slug 必须唯一** — 重复会抛出 `ValueError`
2. **category 必须在 TOOL_CATEGORIES 中定义** — 否则归类到 "other"
3. **文件命名** — `xxx_tool.py` 或 `xxx.py` 均可，只要不是 `__init__.py`
4. **无表单工具** — `form_class = None`，`handle()` 方法中自行处理逻辑
5. **自定义模板** — 设置 `template_name` 为自定义路径，默认为 `tools/tool_detail.html`

## 安全特性

| # | 安全措施 | 实现方式 |
|---|----------|----------|
| 1 | 工具可禁用 | `ToolConfig.is_enabled` 控制，禁用后不可访问 |
| 2 | slug 唯一性 | 发现时检查，重复抛异常 |
| 3 | 表单验证 | 每个工具使用 Django Form 验证输入 |
| 4 | 错误隔离 | 工具模块加载失败不影响其他工具 |
| 5 | 缓存失效保护 | 缓存失效时自动重新发现，无需重启 |
| 6 | 登录保护 | `tool_detail` 与 `my_ip_json` 仅对登录用户开放 |
| 7 | HTTP 工具 SSRF 防护 | `http_request_tool` 会先解析目标域名，只允许公网 IPv4，并把请求固定到已校验 IP |
| 8 | 重定向二次校验 | `http_request_tool` 跟随重定向时会重新解析并校验目标，禁止跳转到内网 |
| 9 | 端口扫描防 DNS Rebinding | `port_scan_tool` 先固定解析结果，再对固定 IP 扫描，避免连接阶段再次解析 |

## 错误处理

| 场景 | 行为 |
|------|------|
| 工具不存在 | 渲染 tool_list.html，显示错误："'{slug}' 不存在" |
| 工具已禁用 | 渲染 tool_list.html，显示错误："'{name}' 已被禁用" |
| my_ip_json 工具不存在 | 返回 404 `{'ok': false, 'error': 'my-ip tool not found'}` |
| 工具模块加载失败 | `logger.warning()` 记录，继续加载其他工具 |
| 工具 slug 重复 | 抛出 `ValueError`（启动时即报错）|
| 工具目录不存在 | 自动创建，下次访问时可加载工具 |
| 工具表单验证失败 | 重新渲染表单，显示错误信息 |
| 工具 handle() 未实现 | 抛出 `NotImplementedError` |
